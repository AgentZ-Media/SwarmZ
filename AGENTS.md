# AGENTS.md

SwarmZ тАФ run and monitor a swarm of native Codex agents (`codex app-server` sessions), organized in project tabs, with an AI Conductor PER PROJECT on top. React frontend + native macOS app (Tauri 2 / Rust). Deep per-subsystem details live in `docs/ARCHITECTURE.md` тАФ read the relevant section there (or the code) before changing a subsystem, and keep it in sync.

**A native Codex-only swarm manager.** SwarmZ is the codex-only application: the old terminal/PTY/grid subsystem, the Claude runtime, the OpenRouter brain and voice dictation were removed in the `rebuild/codex-only-v1` rebuild (git history and `main` preserve the former terminal-grid app; the rebuild plan is `docs/plans/rebuild-codex-only-v1.md`). What exists now is one coherent product:

- **Projects & swarm data model** тАФ sessions organized in per-folder project tabs (canonical-path dedupe), each session a named agent scoped to its project.
- **One Conductor per project** тАФ its own `codex app-server` process slot, its own chats, its own scoped memory and a project-scoped tool arsenal, on the swarm-doctrine operative core.
- **A fully capable engineering lead** тАФ the Conductor decomposes goals onto agents unprompted, steers/reviews them, manages worktrees, sets timers, writes plan documents, and routes routine approvals тАФ a 31-tool arsenal.
- **Genuinely autonomous** тАФ a trigger router turns fleet events (agent finished/blocked, routine-approval escalation, timer, long-idle lanes, PR changes) into budget-gated autonomous Conductor turns; agents deliver `outputSchema`-forced status reports (`expect_report`), finished lanes can auto-review (Settings toggle), and the learning doctrine stores preference observations via `remember`.
- **Vibe v3 UI** тАФ the design-token system + fleet-grid/focus-stage/Conductor-sidebar UI, diffs via @pierre/diffs (`DESIGN.md`).
- **GitHub over the LOCAL `gh` CLI (no OAuth ever, no token)** тАФ read-only detection + panel always; an OPT-IN master toggle arms the Conductor's 7 PR tools, the Rust write gate, the PR watcher (`github://pr-changed` тЖТ Deck ticker + `pr-changed` autonomous turns for watched/auto-review PRs) and the routine-classification of the two sanctioned agent-run gh writes.

New features are native-only Codex; do not reintroduce terminal/PTY/web-mode paths. (Inline `(Phase N)` tags below are historical markers for where a subsystem entered тАФ the app is complete, not mid-rebuild.)

- Quit flushes **all** debounced persists (`flushAllPersists`); the quit/close choreography in `lib/quit.ts` + `lib.rs` looks redundant but every leg is load-bearing (window-state save, updater restart code тАФ the `ExitRequested` handler in `lib.rs` must stay as is).
- Title-bar dragging relies **only** on `data-tauri-drag-region="deep"` on the `<header>` in `TitleBar.tsx` тАФ WKWebView ignores `-webkit-app-region`, so the `drag-region`/`no-drag` CSS classes are no-ops on macOS. `"deep"` makes the whole subtree draggable; Tauri's `drag.js` auto-excludes real `<button>`/`<input>`/`role` elements. The project tabs are plain `<div>`s (activate + close buttons as siblings inside), so they opt out explicitly via `data-tauri-drag-region="false"` тАФ keep that attribute when touching the tab markup. Don't switch back to bare `data-tauri-drag-region` (drags only direct hits on the header itself = just the flex gaps).

## Releases & updates

- Auto-updater polls `latest.json` on GitHub Releases (`lib/updates.ts`; pill in `TitleBar.tsx`, toggle in Settings). About shows `__APP_VERSION__` (Vite define). Builds are ad-hoc signed only.
- Release checklist: bump version in `package.json`, `src-tauri/tauri.conf.json`, `src-tauri/Cargo.toml` (keep in sync) тЖТ create `docs/release-notes/v<X.Y.Z>.md` (CI fails without it) тЖТ tag `v<X.Y.Z>` + push тЖТ `.github/workflows/release.yml` publishes incl. updater manifest.
- Updater signing key: `~/.tauri/swarmz_updater.key` (no password), repo secret `TAURI_SIGNING_PRIVATE_KEY`. Losing it breaks updates for existing installs.

## Ground rules

1. **Never start the dev server** or any long-running app process (`pnpm tauri dev`, `pnpm dev`, тАж) without explicit approval first. Type-checks, builds and `cargo test` are fine.
2. **Never commit or push on your own.** Only when explicitly asked or approved.
3. **Keep docs in sync.** If a change affects anything stated in `AGENTS.md`, `docs/ARCHITECTURE.md` or `README.md`, update them in the same change тАФ proactively, without being asked.
