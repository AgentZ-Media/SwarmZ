# CLAUDE.md

SwarmZ ‚Äî run, tile and monitor multiple Claude Code agents in parallel. React frontend + native macOS app (Tauri 2 / Rust).

**Web mode is abandoned.** The local web mode (`server/` Node engine, `src/lib/backend-web.ts`, `pnpm dev:web`) is not developed anymore: ignore it completely ‚Äî no new `/api/*` endpoints, no parity work, no "keep the two in sync". New features are native-only; remove stale web-mode references from docs when you touch them.

## Stack & commands

- React 19 + TypeScript + Tailwind v4 (dark only ‚Äî follow `DESIGN.md`), Zustand, xterm.js, pnpm
- Type-check: `./node_modules/.bin/tsc --noEmit` ¬∑ Build: `pnpm build` ¬∑ Dev: `pnpm tauri dev`

## Architecture in 10 seconds

- `src/lib/transport.ts` picks the backend at runtime (Tauri vs the abandoned web fallback). Backend capabilities are declared in `backend-types.ts` and implemented in `backend-tauri.ts`. Native-only features may skip the backend interface and use `@tauri-apps/api` directly (e.g. `lib/dnd.ts`).
- `src/store.ts` holds all app state. Persisted: profiles, usage history, workspace tabs (`Workspace`: name/order/defaultCwd + active tab, store key `workspaces`, debounced via `schedulePersistWorkspaces`) + app settings (`AppSettings`: last used folder, default font size, default startup command, claude/git path overrides, auto-update ‚Äî edited in `SettingsDialog.tsx`, opened via the title-bar gear or `‚åò,`) (Tauri store `swarmz.json` / localStorage). Always mutate settings through `updateSettings` (merges + persists the whole object). In-memory only: agents, per-workspace layouts/active panes, focus mode, fleet/palette state. Window size/position persists natively via `tauri-plugin-window-state`. New agents and splits go through the New Agent dialog and land in the active workspace; splits prefill it from the source pane (cwd, profile, startup), plain "new" prefills the configured default startup command (picking a profile overrides it) and cwd from workspace defaultCwd ‚Üí profile default cwd ‚Üí last used folder. The claude path override is applied at PTY spawn (`applyClaudePath` in `utils.ts` ‚Äî replaces a leading `claude` token only).
- **Workspaces** are the top-level containers (tabs in the title bar, `WorkspaceTabs` in `TitleBar.tsx`): each has its own tiling tree (`layouts`/`activePaneIds` keyed by workspace id). Deliberately not bound to a project ‚Äî `defaultCwd` only prefills the New Agent dialog; the first agent with a cwd auto-names the tab (`folderName`, until the user renames it) and sets `defaultCwd`. Tabs show an aggregated status dot + `busy/total` count, rename on double-click, reorder by drag, close via hover-X/middle-click (`CloseWorkspaceDialog.tsx` confirms when agents are inside; closing the last workspace spawns a fresh one ‚Äî there is always ‚â•1). Shortcuts: ‚åò1‚Äì9 switch, ‚åò‚áß[/‚åò‚áß] cycle, ‚åò‚áßN new, ‚åò‚áßW close. Dragging a pane header onto a tab moves the agent there (`moveAgentToWorkspace`; tab highlight via `tabDropTarget`).
- **Terminals live outside React** in `src/lib/term-host.ts` (xterm + PTY per id; `TerminalView` only attaches/detaches the host element). Unmounting a pane detaches the DOM but keeps the terminal running ‚Äî that is what lets panes remount across workspace grids without losing scrollback/PTY. PTYs die exactly on store removal: `removeAgent` / `removeFloatingTerminal` ‚Üí `destroyTerm` (NOT on unmount). The WebGL addon disposes itself on context loss (browser caps live WebGL contexts; with every workspace mounted that's reachable) and xterm falls back to the DOM renderer.
- All workspace grids stay mounted in `WorkspaceLayer.tsx`; inactive ones are `visibility:hidden` + `pointer-events-none` (never unmounted, never resized ‚Äî hidden agents keep working). **Fleet overview** (‚åòE / grid button): the same wrappers are scaled into a card grid with pure CSS transforms (no size change ‚Üí no PTY resize, terminals stay live), with `FleetCard` chrome on top ‚Äî name + status pill, blue pulse on attention, and click-through that maps the cursor onto the exact pane (`paneRects`) and jumps to it. Escape/‚åòE exit (entering blurs the focused terminal so Escape arrives); exit animates via a render-phase `fleetClosing` flag, mirroring the focus-mode pattern.
- **Command palette** (‚åòK, `CommandPalette.tsx`, cmdk): fuzzy-jump to any agent/workspace + all global actions. **‚åò‚áßA** (`attentionJump`) cycles through agents that need input (bell attention or OSC "waiting") across all workspaces.
- Usage is parsed from `~/.claude/projects/*.jsonl` by `src-tauri/src/usage.rs`. Parsing is **incremental**: a per-(file, since) offset cache only reads appended bytes; costs/by_model are recomputed from cached counters on every read (so pricing updates need no cache invalidation). Pricing is fetched live from the OpenRouter model catalog (daily refresh) with a hardcoded per-family fallback for offline; the fallback table in `README.md` must match.
- Subscription limits (5h/weekly windows in the title bar) come from the Anthropic OAuth usage endpoint, authenticated with the local Claude Code login (macOS Keychain `Claude Code-credentials`, fallback `~/.claude/.credentials.json`) ‚Äî implemented in `src-tauri/src/limits.rs`, polled once a minute by `src/lib/limits.ts` (paused while hidden). A `null` result means "no Claude login" and hides the meters; transient fetch errors reject instead, and the store keeps the last known values ‚Äî preserve this distinction. Tokens/cost are tracked but only shown on demand (per-pane stats popover, usage drawer) ‚Äî not in pane headers or the title bar.
- Per-pane git status (repo, branch, ¬±lines, untracked, `origin` URL for "Open repo in browser") is strictly read-only and shells out to the git binary: `src-tauri/src/git.rs` (prefers the Settings `gitPath` override, else `/usr/bin/git` because GUI apps get a minimal PATH). `src/lib/git.ts` polls every 7 s, one fetch per distinct cwd, paused while the window is hidden; `null` on an agent means "checked, not a repo".
- Pane headers are container-query responsive (`@container` on the header in `AgentPane.tsx`): below ~36rem the folder path, gauge readout and split buttons collapse, below ~28rem also the git chip and stats button ‚Äî title, model badge, context donut, focus button and the ‚ãØ menu always remain.
- Focus mode (`focusedAgentId` in the store, in-memory): the pane zooms (~150 ms) to an overlay above a dimmed backdrop in `TilingGrid.tsx` ‚Äî nothing unmounts, all other PTYs keep running underneath. Enter via header button / header double-click / ‚ãØ menu; exit via the same toggle or a backdrop click (no Escape ‚Äî the terminal eats it). Creating or removing agents, switching workspaces and opening the fleet all leave focus mode; pane drag-rearrange is disabled while focused. The exit animation relies on a render-phase `closingFocusId` in `TilingGrid.tsx` that keeps the transition class on the wrapper while the rect snaps back.
- Floating terminals (`FloatingTerminals.tsx`): PiP-style plain shells above the grid, owned by an agent pane (same cwd), opened via header button / ‚ãØ menu / `‚åòJ`. In-memory, rendered in one layer inside `<main>`; windows stay mounted through minimize/detach and are only hidden (not unmounted) while their owner's workspace is inactive or the fleet is open ‚Äî removal from the store is what kills the PTY (`destroyTerm`); detached windows (`agentId: null`) show in every workspace. Stacking via a per-window `z` (never reorder the DOM ‚Äî xterm canvas). Windows name themselves after the last command run ‚Äî preset clicks and typed lines alike (`onCommand` on `TerminalView`, a best-effort input-line buffer that skips history-recall/escape-edited lines). Their quick-command bar mixes **presets per project folder** (persisted, store key `commandPresets` ‚Üí `FolderCommands { presets, hidden }`, keyed by cwd via `presetKey`; presets are editable in place, a preset matching a detected command's label or command **overrides** it, and detected commands can be hidden per folder + restored) with auto-detected commands: `project_commands` in `src-tauri/src/project.rs` (package.json scripts incl. lockfile‚Üípackage-manager mapping, cargo, Makefile/justfile targets; script order is alphabetical). Closing a pane goes through `requestRemoveAgent`, which asks the backend whether a floating terminal still has child processes (`pty_has_children`, pgrep on the shell pid) and only then raises the kill-vs-detach dialog (`CloseAgentDialog.tsx`); detached terminals live on with `agentId: null`.
- Pane title & live status come from terminal escape sequences, handled in `Terminal.tsx`: OSC 0/2 titles (claude's auto-generated topic) rename the agent unless the user renamed it manually (`renamed` flag; clearing the name re-enables auto-titles), OSC 9;4 progress reporting maps to `Agent.activity` busy/idle (the status dot, the live per-tab `busy/total` counts, the fleet cards, and the quit guard). Claude Code only emits OSC 9;4 if the terminal advertises support ‚Äî the PTY backend sets `ConEmuANSI=ON` for that (deliberately, least invasive marker; don't remove). OSC 21337 (claude's idle/busy/waiting tab-status pill) is pre-wired but dormant ‚Äî current Claude Code builds never emit it.
- Quit guard (`src/lib/quit.ts`, started in `App.tsx`): closing the app while any agent is `activity === "busy"` raises `QuitConfirmDialog.tsx` listing the working agents. Native covers both paths ‚Äî window close via `onCloseRequested` (preventDefault + dialog; confirming re-issues `close()` past a module-level `confirmed` flag so the window-state plugin still saves), and ‚åòQ/menu quit via Rust (`lib.rs` run-callback: `ExitRequested` with `code.is_some()` ‚Üí `prevent_exit()` + emit `app://quit-requested`; `prevent_exit` is a tauri-built-in no-op for the updater's `RESTART_EXIT_CODE`, and code `None` = last window closed passes, which is how a confirmed quit actually exits).
- OS file drag & drop onto terminals: Tauri's default `dragDropEnabled` intercepts file drags at the NSView level, so the DOM never sees HTML5 drop events ‚Äî `src/lib/dnd.ts` (started in `App.tsx`) subscribes to the webview's `onDragDropEvent`, hit-tests the cursor via `elementFromPoint` against `[data-file-drop]` zones (the terminal containers in `AgentPane.tsx` and `FloatingTerminals.tsx`, attribute value = pty id) and on drop **pastes** the Terminal.app-style backslash-escaped path(s) (`escapeDropPath` in `utils.ts`) + a trailing space into that terminal via `term.paste()` (xterm instances are looked up in `lib/term-registry.ts`, registered by `TerminalView`), then focuses/raises the target. Paste, not a raw PTY write, is essential: `term.paste()` wraps the text in bracketed-paste markers when the app enabled mode 2004, and Claude Code only attaches image paths that arrive as a paste ‚Äî raw writes count as keystrokes and stay plain text. While a drag is in flight, every zone shows `FileDropOverlay.tsx` (store key `fileDrag`, in-memory; `pointer-events-none` so the overlay stays invisible to the hit test); the hovered zone lights up `--ring` blue. Beware: the drag events' `position` is typed `PhysicalPosition`, but on macOS the wry ‚Üí tauri chain never scales NSView points, so the values are effectively logical CSS px ‚Äî use them unscaled.
- Perf invariants: PTY output is coalesced (‚â§12 ms / 128 KiB) and addressed per agent (`pty://data/<id>`); the usage file-watcher emits the changed project-dir names and the frontend skips refreshes for dirs it doesn't display, throttles to ‚â•2 s, and pauses while the window is hidden.

## Releases & updates

- In-app auto-updater (`tauri-plugin-updater`) polls `latest.json` on GitHub Releases; frontend logic in `src/lib/updates.ts`, UI: status pill in `TitleBar.tsx`, manual check + auto-download toggle (`settings.autoUpdate` ‚Äî downloads on discovery, installs on restart) in `SettingsDialog.tsx`. The About panel shows `__APP_VERSION__` (injected from `package.json` via Vite `define`). Builds are ad-hoc signed only (no Apple notarization).
- Release checklist: bump version in `package.json`, `src-tauri/tauri.conf.json` and `src-tauri/Cargo.toml` (keep in sync) ‚Üí create `docs/release-notes/v<X.Y.Z>.md` (CI fails without it) ‚Üí tag `v<X.Y.Z>` and push the tag ‚Üí `.github/workflows/release.yml` builds macOS (Apple Silicon) and publishes the release incl. updater manifest.
- Updater signing key: `~/.tauri/swarmz_updater.key` (no password), repo secret `TAURI_SIGNING_PRIVATE_KEY`. Losing it breaks updates for existing installs.
