# CLAUDE.md

SwarmZ — run, tile and monitor multiple Claude Code agents in parallel. React frontend + native macOS app (Tauri 2 / Rust).

**Web mode is abandoned.** The local web mode (`server/` Node engine, `src/lib/backend-web.ts`, `pnpm dev:web`) is not developed anymore: ignore it completely — no new `/api/*` endpoints, no parity work, no "keep the two in sync". New features are native-only; remove stale web-mode references from docs when you touch them.

## Stack & commands

- React 19 + TypeScript + Tailwind v4 (dark only — follow `DESIGN.md`), Zustand, xterm.js, pnpm
- Type-check: `./node_modules/.bin/tsc --noEmit` · Build: `pnpm build` · Dev: `pnpm tauri dev`

## Architecture in 10 seconds

- `src/lib/transport.ts` picks the backend at runtime (Tauri vs the abandoned web fallback). Backend capabilities are declared in `backend-types.ts` and implemented in `backend-tauri.ts`. Native-only features may skip the backend interface and use `@tauri-apps/api` directly (e.g. `lib/dnd.ts`).
- `src/store.ts` holds all app state. Persisted: profiles, usage history, workspace tabs (`Workspace`: name/order/defaultCwd + active tab, store key `workspaces`, debounced via `schedulePersistWorkspaces`), the grid snapshot for session restore (store key `grid`, see below) + app settings (`AppSettings`: last used folder, default font size, default startup command, claude/git path overrides, auto-update, restore-on-launch, voice-dictation prefs (engine cloud/local, hotkey mode, auto-submit, cleanup toggle/model/prompt, STT model) — edited in `SettingsDialog.tsx`, opened via the title-bar gear or `⌘,`) (Tauri store `swarmz.json` / localStorage). Always mutate settings through `updateSettings` (merges + persists the whole object). In-memory only: floating terminals, focus mode, fleet/palette state. Window size/position persists natively via `tauri-plugin-window-state`. New agents and splits go through the New Agent dialog and land in the active workspace; splits prefill it from the source pane (cwd, profile, startup), plain "new" prefills the configured default startup command (picking a profile overrides it) and cwd from workspace defaultCwd → profile default cwd → last used folder. The claude path override is applied at PTY spawn (`applyClaudePath` in `utils.ts` — replaces a leading `claude` token only).
- **Session restore**: the live grid is snapshotted continuously (store key `grid`, debounced `schedulePersistGrid` in `store.ts`: each agent's restore-relevant fields incl. the latched claude `sessionId`, plus the per-workspace layout trees and active panes) and flushed on every quit path (`flushPersistGrid`, called in `lib/quit.ts` before the window actually closes). On launch `hydrate()` rebuilds panes from the snapshot only if `settings.restoreAgents === true` (default **off**) and no agent was already spawned; torn snapshots degrade gracefully (panes without their agent are pruned, agents without a pane are dropped). Restored agents carry the transient `Agent.resume = sessionId` — applied only at PTY spawn (`Terminal.tsx` → `resumeStartup` in `utils.ts`, appends `--resume <id>` to a leading `claude` command; compound commands and explicit `--resume/--continue` pass through), so `agent.startup` stays clean for split-prefill and re-persisting. `claude --resume` keeps the same session id, so usage latches right back onto the same jsonl — and a latched session is parsed without the since-filter (`usage_for_session` in `usage.rs`), so stats and the context donut include the pre-restart turns despite the reset `createdAt`. Session latching is gated on real activity: an unlatched pane only runs session discovery once its claude first reported busy (`Agent.firstBusyAt`, stamped in `setActivity` from OSC 9;4), and discovery then only matches files born around that first activity (App.tsx usage refresh) — an idle pane never latches (and never resumes) a sibling session born in the same folder. Floating terminals are not captured.
- **Workspaces** are the top-level containers (tabs in the title bar, `WorkspaceTabs` in `TitleBar.tsx`): each has its own tiling tree (`layouts`/`activePaneIds` keyed by workspace id). Deliberately not bound to a project — `defaultCwd` only prefills the New Agent dialog; the first agent with a cwd auto-names the tab (`folderName`, until the user renames it) and sets `defaultCwd`. Tabs show an aggregated status dot + `busy/total` count, rename on double-click, reorder by drag, close via hover-X/middle-click (`CloseWorkspaceDialog.tsx` confirms when agents are inside; closing the last workspace spawns a fresh one — there is always ≥1). Shortcuts: ⌘1–9 switch, ⌘⇧[/⌘⇧] cycle, ⌘⇧N new, ⌘⇧W close. Dragging a pane header onto a tab moves the agent there (`moveAgentToWorkspace`; tab highlight via `tabDropTarget`).
- **Workspace presets** (store key `workspacePresets`, helpers in `lib/presets.ts`): named layout trees whose leaves are pane templates (`PresetPaneNode`: cwd unset = *inherit* the folder asked once at load via `LoadPresetDialog.tsx` — skipped when every pane is pinned; startup unset = default startup command, `""` = plain shell; plus name/profileId/color). Seeded with Solo/1×2/2×2/1+2 only when the key is **missing** — an emptied list stays empty. Loaded from the empty-workspace screen (`EmptyState` in `WorkspaceLayer.tsx`, cards with `PresetThumbnail.tsx` mini-grids; ⌘⇧N → empty tab → cards covers the new-workspace flow): `applyPreset` builds all agents + the live layout in one pass, fills the active workspace if still empty (else a new tab) and names the tab after the preset (`renamed: true`). Created via ⌘K → "Save workspace as preset" (`SavePresetDialog.tsx` → `presetLayoutFromGrid` snapshots the live grid; agent names only carry over when user-renamed). Managed in Settings → Presets (rename, per-pane folder/command, remove panes).
- **Terminals live outside React** in `src/lib/term-host.ts` (xterm + PTY per id; `TerminalView` only attaches/detaches the host element). Unmounting a pane detaches the DOM but keeps the terminal running — that is what lets panes remount across workspace grids without losing scrollback/PTY. PTYs die exactly on store removal: `removeAgent` / `removeFloatingTerminal` → `destroyTerm` (NOT on unmount). The WebGL addon disposes itself on context loss (browser caps live WebGL contexts; with every workspace mounted that's reachable) and xterm falls back to the DOM renderer.
- All workspace grids stay mounted in `WorkspaceLayer.tsx`; inactive ones are `visibility:hidden` + `pointer-events-none` (never unmounted, never resized — hidden agents keep working). **Fleet overview** (⌘E / grid button): the same wrappers are scaled into a card grid with pure CSS transforms (no size change → no PTY resize, terminals stay live), with `FleetCard` chrome on top — name + status pill, blue pulse on attention, and click-through that maps the cursor onto the exact pane (`paneRects`) and jumps to it. Escape/⌘E exit (entering blurs the focused terminal so Escape arrives); exit animates via a render-phase `fleetClosing` flag, mirroring the focus-mode pattern.
- **Command palette** (⌘K, `CommandPalette.tsx`, cmdk): fuzzy-jump to any agent/workspace + all global actions; custom commands surface as their own group **only while a search is typed** (folder commands of the target pane + globals) and run with the same paste/⌘↵ semantics as ⌘⇧K. **⌘⇧A** (`attentionJump`) cycles through agents that need input (bell attention or OSC "waiting") across all workspaces.
- **Custom commands** (⌘⇧K, `InsertCommandPalette.tsx`): user-saved prompt snippets, global + per folder (store key `customCommands` → `CustomCommandsData`, folder lists keyed via `presetKey` — deliberately separate from the floating-terminal `commandPresets`, which are executed shell commands). Selecting **pastes** into the target pane via `term.paste()` without submitting; ⌘Enter submits via a **separate** `ptyWrite "\r"` after the paste (a `\r` inside the bracketed paste is only a literal newline in claude's input box) — the paste/submit core is shared with ⌘K in `lib/insert-command.ts` (`insertCommandText`); a `{{input}}` command picked in ⌘K detours through the picker's form via store `commandPickerPreselect` (in-memory, consumed on open). Target = focused agent, else active pane; agent panes only (floating terminals have no focus tracking in the store — extending later is just target resolution, `insert()` only needs a pty id). Placeholders live in `lib/command-vars.ts`: `{{folder}}/{{cwd}}/{{branch}}/{{agent}}` fill from the pane, `{{input:Label}}` asks inline before inserting (same label twice = same value), unknown `{{…}}` stay literal. Managed in Settings → Commands, quick-add directly in the picker. The ⌘⇧K keydown branch in `App.tsx` must stay **before** the plain ⌘K branch (which doesn't check shift).
- **Voice dictation** (hold mode = plain ⌘ push-to-talk, toggle mode = ⌘⇧M, per `dictationHotkeyMode`; or the mic button in pane/floating headers). Hold mode arms on Meta keydown and only opens the mic after ~250 ms held **alone** (`armHoldDictation` — ⌘-shortcuts never trigger it; any other keydown cancels silently via `cancelHoldDictation`), Meta keyup or window blur finishes (`finishHoldDictation`); recordings under ~1 s are discarded instead of transcribed (OpenRouter rejects near-empty audio). recording happens in the webview (`lib/dictation.ts`, outside React like term-host — getUserMedia + MediaRecorder, rotated into ~50 s segments because the OpenRouter transcription API wants ≤60 s uploads, 5 min hard cap; the AnalyserNode feeds the waveform pill in `Dictation.tsx`, rendered inside the target's terminal container). Two engines (`settings.dictationEngine`, default `"openrouter"`): cloud transcription, optional LLM cleanup and the model catalog run natively in `src-tauri/src/openrouter.rs`; the **local engine** (`"local"`, `src-tauri/src/localstt.rs` + `lib/local-stt.ts`) runs NVIDIA Parakeet TDT 0.6b v3 int8 on-device via transcribe-rs/ONNX Runtime (statically linked at build time through ort's `download-binaries` — no Python or system deps ever) — the ~670 MB model (4 files from HF `istupakov/parakeet-tdt-0.6b-v3-onnx`) is downloaded once from Settings → Voice into `~/Library/Application Support/SwarmZ/models/` (`.part` + rename, resumable, progress via `localstt://progress`), loaded lazily on first dictation (~1.5–2 GB RAM, cached until unload — switching the engine back to cloud unloads it) and fed 16 kHz mono PCM that the webview decodes/resamples from the recording via OfflineAudioContext (`blobToPcm16k`). Mic gating is engine-aware (`selectDictationReady`: local → store `localSttStatus.installed`, cloud → `openrouterStatus`); the cleanup pass always goes through OpenRouter, so with the local engine it only runs when a usable key is also stored. the API key lives in the macOS Keychain (service `SwarmZ-OpenRouter`, read/written via the `security` CLI like limits.rs) and never reaches JS — `lib/openrouter.ts` is native-only direct `invoke` (skips the backend interface, like dnd.ts). The transcript is **pasted** like the insert picker (`term.paste`, separate `\r` only with `dictationAutoSubmit`). Cleanup uses structured output (`json_schema {cleaned}` + `require_parameters`) with `reasoning: {effort: "minimal"}` — NOT `enabled: false`, Gemini 400s on that ("Reasoning is mandatory … cannot be disabled"); models that reject "minimal" get one retry without the reasoning field — and a user-editable prompt whose default forbids translating — dictation must stay language-agnostic. All mic UI is gated on store `openrouterStatus` (fetched at launch and on key save; `valid: null` = present-but-unverified counts as usable, only a 401/403 hides it). One dictation at a time (store `dictation`, in-memory). Mic permission comes from `src-tauri/Info.plist` (`NSMicrophoneUsageDescription`).
- Usage is parsed from `~/.claude/projects/*.jsonl` by `src-tauri/src/usage.rs`. Parsing is **incremental**: a per-(file, since) offset cache only reads appended bytes; costs/by_model are recomputed from cached counters on every read (so pricing updates need no cache invalidation). Pricing is fetched live from the OpenRouter model catalog (daily refresh) with a hardcoded per-family fallback for offline; the fallback table in `README.md` must match.
- Subscription limits (5h/weekly windows in the title bar) come from the Anthropic OAuth usage endpoint, authenticated with the local Claude Code login (macOS Keychain `Claude Code-credentials`, fallback `~/.claude/.credentials.json`) — implemented in `src-tauri/src/limits.rs`, polled once a minute by `src/lib/limits.ts` (paused while hidden). A `null` result means "no Claude login" and hides the meters; transient fetch errors reject instead, and the store keeps the last known values — preserve this distinction. Tokens/cost are tracked but only shown on demand (per-pane stats popover, usage drawer) — not in pane headers or the title bar.
- Per-pane git status (repo, branch, ±lines, untracked, `origin` URL for "Open repo in browser") is strictly read-only and shells out to the git binary: `src-tauri/src/git.rs` (prefers the Settings `gitPath` override, else `/usr/bin/git` because GUI apps get a minimal PATH). `src/lib/git.ts` polls every 7 s, one fetch per distinct cwd, paused while the window is hidden; `null` on an agent means "checked, not a repo".
- Pane headers are container-query responsive (`@container` on the header in `AgentPane.tsx`): below ~36rem the folder path, gauge readout and split buttons collapse, below ~32rem the model badge, below ~28rem also the git chip, stats, mic and focus buttons — only status dot, truncated title (`min-w-0 truncate`, never `shrink-0` — it must shrink for the ellipsis to engage), context donut, ⋯ menu and close remain. Everything hidden stays reachable via the ⋯ menu or hotkeys (focus = header double-click, dictation = ⌘ hold/⌘⇧M).
- Focus mode (`focusedAgentId` in the store, in-memory): the pane zooms (~150 ms) to an overlay above a dimmed backdrop in `TilingGrid.tsx` — nothing unmounts, all other PTYs keep running underneath. Enter via header button / header double-click / ⋯ menu; exit via the same toggle or a backdrop click (no Escape — the terminal eats it). Creating or removing agents, switching workspaces and opening the fleet all leave focus mode; pane drag-rearrange is disabled while focused. The exit animation relies on a render-phase `closingFocusId` in `TilingGrid.tsx` that keeps the transition class on the wrapper while the rect snaps back.
- Floating terminals (`FloatingTerminals.tsx`): PiP-style plain shells above the grid, owned by an agent pane (same cwd), opened via header button / ⋯ menu / `⌘J`. In-memory, rendered in one layer inside `<main>`; windows stay mounted through minimize/detach and are only hidden (not unmounted) while their owner's workspace is inactive or the fleet is open — removal from the store is what kills the PTY (`destroyTerm`); detached windows (`agentId: null`) show in every workspace. Stacking via a per-window `z` (never reorder the DOM — xterm canvas). Windows name themselves after the last command run — preset clicks and typed lines alike (`onCommand` on `TerminalView`, a best-effort input-line buffer that skips history-recall/escape-edited lines). Their quick-command bar mixes **presets per project folder** (persisted, store key `commandPresets` → `FolderCommands { presets, hidden }`, keyed by cwd via `presetKey`; presets are editable in place, a preset matching a detected command's label or command **overrides** it, and detected commands can be hidden per folder + restored) with auto-detected commands: `project_commands` in `src-tauri/src/project.rs` (package.json scripts incl. lockfile→package-manager mapping, cargo, Makefile/justfile targets; script order is alphabetical). Closing a pane goes through `requestRemoveAgent`, which asks the backend whether a floating terminal still has child processes (`pty_has_children`, pgrep on the shell pid) and only then raises the kill-vs-detach dialog (`CloseAgentDialog.tsx`); detached terminals live on with `agentId: null`.
- Pane title & live status come from terminal escape sequences, handled in `Terminal.tsx`: OSC 0/2 titles (claude's auto-generated topic) rename the agent unless the user renamed it manually (`renamed` flag; clearing the name re-enables auto-titles), OSC 9;4 progress reporting maps to `Agent.activity` busy/idle (the status dot, the live per-tab `busy/total` counts, the fleet cards, and the quit guard). Claude Code only emits OSC 9;4 if the terminal advertises support — the PTY backend sets `ConEmuANSI=ON` for that (deliberately, least invasive marker; don't remove). OSC 21337 (claude's idle/busy/waiting tab-status pill) is pre-wired but dormant — current Claude Code builds never emit it.
- Quit guard (`src/lib/quit.ts`, started in `App.tsx`): closing the app raises `QuitConfirmDialog.tsx` when quitting would lose something (`quitBlockerIds`) — agents with `activity === "busy"` always count (the run gets interrupted, spinner in the list); idle-but-open panes count only while `restoreAgents` is off (with restore on they come back anyway, so they don't block). Native covers both paths — window close via `onCloseRequested` (every first close is preventDefault'ed, the grid snapshot is flushed, then `close()` is re-issued past a module-level `confirmed` flag so the window-state plugin still saves), and ⌘Q/menu quit via Rust (`lib.rs` run-callback: `ExitRequested` with `code.is_some()` → `prevent_exit()` + emit `app://quit-requested`; `prevent_exit` is a tauri-built-in no-op for the updater's `RESTART_EXIT_CODE`, and code `None` = last window closed passes, which is how a confirmed quit actually exits).
- OS file drag & drop onto terminals: Tauri's default `dragDropEnabled` intercepts file drags at the NSView level, so the DOM never sees HTML5 drop events — `src/lib/dnd.ts` (started in `App.tsx`) subscribes to the webview's `onDragDropEvent`, hit-tests the cursor via `elementFromPoint` against `[data-file-drop]` zones (the terminal containers in `AgentPane.tsx` and `FloatingTerminals.tsx`, attribute value = pty id) and on drop **pastes** the Terminal.app-style backslash-escaped path(s) (`escapeDropPath` in `utils.ts`) + a trailing space into that terminal via `term.paste()` (xterm instances are looked up in `lib/term-registry.ts`, registered by `TerminalView`), then focuses/raises the target. Paste, not a raw PTY write, is essential: `term.paste()` wraps the text in bracketed-paste markers when the app enabled mode 2004, and Claude Code only attaches image paths that arrive as a paste — raw writes count as keystrokes and stay plain text. While a drag is in flight, every zone shows `FileDropOverlay.tsx` (store key `fileDrag`, in-memory; `pointer-events-none` so the overlay stays invisible to the hit test); the hovered zone lights up `--ring` blue. Beware: the drag events' `position` is typed `PhysicalPosition`, but on macOS the wry → tauri chain never scales NSView points, so the values are effectively logical CSS px — use them unscaled.
- Perf invariants: PTY output is coalesced (≤12 ms / 128 KiB) and addressed per agent (`pty://data/<id>`); the usage file-watcher emits the changed project-dir names and the frontend skips refreshes for dirs it doesn't display, throttles to ≥2 s, and pauses while the window is hidden.

## Releases & updates

- In-app auto-updater (`tauri-plugin-updater`) polls `latest.json` on GitHub Releases; frontend logic in `src/lib/updates.ts`, UI: status pill in `TitleBar.tsx`, manual check + auto-download toggle (`settings.autoUpdate` — downloads on discovery, installs on restart) in `SettingsDialog.tsx`. The About panel shows `__APP_VERSION__` (injected from `package.json` via Vite `define`). Builds are ad-hoc signed only (no Apple notarization).
- Release checklist: bump version in `package.json`, `src-tauri/tauri.conf.json` and `src-tauri/Cargo.toml` (keep in sync) → create `docs/release-notes/v<X.Y.Z>.md` (CI fails without it) → tag `v<X.Y.Z>` and push the tag → `.github/workflows/release.yml` builds macOS (Apple Silicon) and publishes the release incl. updater manifest.
- Updater signing key: `~/.tauri/swarmz_updater.key` (no password), repo secret `TAURI_SIGNING_PRIVATE_KEY`. Losing it breaks updates for existing installs.

## Ground rules

1. **Never start the dev server** or any long-running app process (`pnpm tauri dev`, `pnpm dev`, …) without explicit approval first. Type-checks and builds that don't launch the app are fine.
2. **Never commit or push on your own.** Only when explicitly asked or approved.
3. **Keep docs in sync.** If a change affects anything stated in `CLAUDE.md` or `README.md`, update them in the same change — proactively, without being asked.
