# v0.4.0 — The Orchestrator & Mission Control

The biggest release so far: an AI team lead over your agent fleet, and a
signal-architecture redesign built for running many agents at once.

## Orchestrator (⌘⇧O)

- New chat sidebar housing an AI orchestrator with full fleet context: it
  sees every pane (status, project, branch, transcripts), reads project docs
  and quick notes, and knows every project on the machine.
- It acts through tools: prompt any pane (style-adapted per runtime — direct
  orders for Codex, goal + context for Claude), create panes and workspaces,
  optionally in fresh git worktrees ("four agents, four worktrees" is one
  request), pick models per pane (`--model` / `-m`, with recently used model
  ids suggested from real usage) — or omit everything and your defaults apply.
- Status pings: when an agent the orchestrator prompted finishes or asks for
  input, the chat shows it — and feeds it back into the model's context, so
  "great, continue" just works.
- Two brains, switchable in Settings → Orchestrator: Codex app-server
  (default, uses your ChatGPT login; dynamic tools, streaming, interrupt,
  thread resume across restarts) or an OpenRouter tool loop with free model
  choice. Auto-submit can be turned off (review mode), busy panes can be
  refused, and each app launch starts a fresh chat (history stays).

## Mission Control redesign

- Signal triad, coupled color + shape + word everywhere: blue now exclusively
  means "where I am" (focus/selection), the new amber means "where I'm
  needed" (`⚑ needs you` header tint + one-shot arrival flash), green marks
  "just finished" (fades after 5 min) — and busy is quiet: a subtle activity
  hairline instead of pulsing dots.
- The Deck: a permanent bottom status bar with a triage queue ("⚑ 2 need
  you" with jump-to-pane popover), a live fleet event ticker (finished /
  waiting / orchestrator prompts / created / exited), the usage meters
  (moved out of the title bar) and orchestrator status.
- Codex limit meters are now account-level — read from your newest session
  data regardless of open panes, with an honest `CX —` placeholder and
  staleness tooltip instead of silently disappearing.
- Fleet view (⌘E) became an ops board: cards carry status, project · branch,
  last event with age and a context bar — fully keyboard-operable (arrows,
  Tab cycles panes that need you, Home/End, Enter to jump in).
- Keyboard & accessibility: visible focus rings on every control,
  keyboard-reachable close/delete buttons, focusable tooltips, new ⌘] / ⌘[
  pane cycling, and a global reduced-motion mode honoring the OS setting.
- Polish: one product language (English), human-readable worktree errors,
  clearer meter tooltips, quieter workspace tabs (amber `⚑n` instead of
  pulsing), documented z-scale and design-system updates in DESIGN.md.
