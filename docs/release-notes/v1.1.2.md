# v1.1.2 — Observable worker delivery

SwarmZ 1.1.2 makes multi-worker startup honest and worker conversations much
easier to follow. It also fixes two persistent interface annoyances around
narrow Orchestrator headers and historical GitHub failures.

## Reliable agent startup

- Parallel batches now require separate worktrees for independent agents.
  Multiple workers targeting the same main checkout are rejected before any
  empty worker window is created, with an explicit parallel/sequential retry
  instruction.
- A worker is reported as started only after its initial `turn/start` receives
  a native backend acknowledgement. Spawn summaries distinguish created
  sessions from acknowledged assignments and expose the bound turn id.
- The Orchestrator manual now tells the lead to use one fresh worktree per
  independent lane and never infer activity from a visible worker window.

## Human-readable worker chats

- Agent and Orchestrator messages share one safe GitHub-Flavored Markdown
  renderer with tables, task lists, code, headings, links and strikethrough.
  Raw HTML stays inert, and external links keep the native opener boundary.
- Markdown is rendered during the existing batched message stream, so long
  audit responses no longer expose headings, bold markers or table pipes as
  raw text while the worker is writing.
- Consecutive commands and web searches collapse into a compact
  `Working/Worked · N steps` activity row. Exact commands and bounded output
  remain available on demand, while Fleet cards prioritize agent prose.
  Expanded activity rows render real file-change events as syntax-highlighted
  diffs instead of reducing edits to their underlying command.
- Schema-shaped `{done:false,...}` progress updates render as readable status
  cards. Only final backend-stamped reports retain Mission/evidence authority.

## Interface fixes

- A new default-on **Attention sound** setting plays one quiet Web Audio cue
  when an agent newly starts waiting for human input. It never replays
  hydrated historic blockers and can be disabled under Appearance.
- The resizable Orchestrator header now degrades cleanly at narrow widths:
  the chat title owns the ellipsis slot, model controls become icon-only and
  every remaining control stays fully visible instead of being clipped.
- GitHub CI, conflict and review failures are one-shot attention notices.
  Opening the Attention Inbox marks the exact observed revisions as read;
  changed or newly failing revisions appear again, while the real GitHub state
  remains untouched. Worker decisions and Mission blockers cannot be dismissed.
