# v0.3.8 — updater: no more "frozen" Check now button

Clicking **Check now** after an update was already found or downloaded used to
do nothing — silently, with no feedback — which felt like the app had hung.
That dead state is gone.

## 🔄 Update controls behave predictably

- Once an update is **available**, **downloading** or **downloaded (ready to
  restart)**, the **Check now** button is now disabled. In those states the
  only sensible action is **Download** or **Restart now**, so re-checking is no
  longer offered.
- Under the hood the manual check also bails cleanly in those states instead of
  re-entering the updater while it still holds a downloaded artifact, and it
  stops the spinner immediately — a click can never look like a freeze again.
