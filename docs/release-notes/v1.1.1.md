# v1.1.1 — Mission Control

SwarmZ 1.1.1 is the production-ready Mission Control release for large Codex
engineering goals. It supersedes the withdrawn 1.1.0 binary and includes the
complete Mission Control feature set plus a corrected macOS app-bundle
signature.

## Durable Missions

- Persist tasks, dependencies, attempts, decisions, artifacts, quality gates,
  schedules, reminders and a replayable event audit.
- Schedule fresh task-bound workers against live capacity with exact-base Git
  worktrees, conflict prevention and a human-approved Mission Envelope.
- Recover safely from crashes through durable outbox fencing, reconciliation
  and a visible recovery ledger.

## One Orchestrator, temporary workers

- Each project has one fixed learning Orchestrator with transparent scoped
  memory.
- Persona presets, the personality editor and reusable worker identities are
  removed.
- Workers are temporary one-assignment attempts; their evidence stays with the
  Mission, but no personality or durable worker memory is reused.

## Evidence and integration

- Native Git state, scoped file changes, exact command exits, reviews and usage
  evidence decide success instead of worker self-reports.
- Dependency-ordered Integration Trains expose checkpoints plus human-driven
  Retry, Skip and Rollback recovery.
- Optional Runtime Environments provide direct-argv setup and cleanup, owned
  local services, deterministic ports/database namespaces and a no-network
  macOS acceptance sandbox. Ordinary Missions work fully without a runtime.

## Large-task workflows and interface

- Import Markdown, plain text, CSV, JSON and GitHub Issues.
- Use playbooks, temporary task roles, multi-root work and candidate A/B/N
  attempts without creating persistent worker personas.
- Work from Board, dependency Graph, Fleet, Integration and Timeline views,
  with task inspection, global Attention Inbox, Insights, archive/search,
  runtime and recovery surfaces.
- Accessibility, reduced motion, keyboard operation, loading/empty states,
  responsive layouts and contrast were audited in the real macOS Dev app.

## Packaging correction

The short-lived 1.1.0 download was removed before handoff after release QA
found that macOS had signed only the executable, not the complete `.app`
bundle. Version 1.1.1 uses Tauri's explicit ad-hoc signing identity, contains
`Contents/_CodeSignature/CodeResources`, preserves the stable bundle identifier
`de.agent-z.swarmz` and passes `codesign --verify --deep --strict`.

The release pipeline now creates a private draft, verifies the complete app
signature and DMG integrity, and publishes only after those checks pass.
