# v1.1.0 — Mission Control

SwarmZ is now a native Mission Control for large engineering goals. Instead
of treating a long backlog as chat context, it turns the work into durable,
bounded and independently verifiable product state.

## Durable Missions instead of transient coordination

- Missions persist tasks, dependencies, attempts, decisions, artifacts,
  quality gates, schedules, reminders and a replayable event audit.
- The adaptive scheduler admits work against live backend capacity, freezes a
  human-approved Mission Envelope and prevents overlapping writers.
- Crash-safe outbox fencing, restart reconciliation and visible recovery keep
  interrupted work explainable instead of silently restarting it.

## One Orchestrator, temporary workers

- Every project has one fixed Orchestrator identity with transparent scoped
  memory and learning.
- Persona presets, the personality editor and legacy reusable-agent identity
  paths are removed.
- Every worker is a fresh, task-bound attempt with no persona or durable
  memory. Its evidence remains with the Mission; the worker is retired.

## Evidence, runtimes and integration

- Success is decided from native Git state, scoped file changes, exact command
  exit codes, review and usage evidence — never from a worker's self-report.
- Runtime Environments provide direct-argv setup and cleanup, local services,
  port and database leases, and a no-network macOS acceptance sandbox.
- Dependency-ordered Integration Trains expose durable checkpoints and
  human-controlled Retry, Skip and Rollback recovery.

## Large-task workflows

- Import Markdown, text, CSV, JSON and GitHub Issues into a Mission.
- Reuse playbooks and temporary task roles without creating persistent worker
  personalities.
- Coordinate multi-root work, candidate A/B/N attempts, bounded cost and ETA
  insights, PR monitoring and Orchestrator-created pull requests.

## Mission Control interface

- A complete Mission workspace provides Board, dependency Graph, Fleet,
  Integration and Timeline views plus a task inspector, Attention Inbox,
  runtime drawer, recovery ledger, archive/search flows and Mission Insights.
- Dense states, empty states, loading, keyboard navigation, reduced motion,
  responsive minimum-window layouts and WCAG-AA contrast were audited as part
  of the release.

## Security, persistence and maintainability

- Persistence is fail-closed and serialized across Missions, outbox, runtime,
  projects, sessions, chats, timers, notes, settings and autonomy budgets.
- Filesystem, process, worktree, GitHub, approval and sandbox boundaries were
  hardened and covered by frontend and Rust regression tests.
- Mission lifecycle, Orchestrator delivery/watchers, tool executors and native
  test suites were split into explicit services with stable ownership.

## Upgrade notes

Schema v3 removes the legacy persisted persona setting while preserving
projects, sessions, chats and scoped Orchestrator memory. Mission, outbox and
Runtime Environment state continues to live locally in SwarmZ's application
store.
