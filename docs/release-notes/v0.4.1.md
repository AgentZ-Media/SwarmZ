# v0.4.1 — Orchestrator startup fix

- Fixed the orchestrator failing with "failed to start `codex app-server`:
  No such file or directory" in the packaged app. macOS gives GUI apps a
  minimal PATH, so the bare `codex` command that worked in dev could not be
  found once installed. The Codex binary is now resolved to an absolute path
  (Settings override → PATH → well-known install locations like
  `~/.local/bin` and `/opt/homebrew/bin` → a login-shell probe covering
  nvm/asdf/mise setups), cached, and the app-server child process inherits
  an enriched PATH so tools it spawns by name resolve too.
