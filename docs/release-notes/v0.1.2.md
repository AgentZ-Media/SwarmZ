## SwarmZ v0.1.2 — plan limits, smarter dialogs & window memory 🧠

### New

- **Plan limits in the title bar** 📈 — SwarmZ now shows the usage limits of
  the Claude subscription logged into Claude Code on this machine: the
  5-hour session window and the weekly windows, each as a small meter with
  the reset time on hover. Authenticated with your local Claude Code login;
  refreshed once a minute. (macOS may ask once for Keychain access — choose
  "Always Allow".)
- **Per-agent stats popover** — a stats button in every pane header opens
  everything tracked for that agent on demand: context occupancy, tokens
  (input / output / cache), messages, estimated API cost, branch, folder and
  start time.
- **Splitting opens the agent dialog** — the split buttons (and `⌘D` /
  `⌘⇧D`) now open the New Agent dialog prefilled with the source pane's
  folder, profile and startup command instead of silently spawning in the
  default directory.
- **Folder memory** — the New Agent dialog prefills the profile's default
  folder, or the last folder you launched an agent in (persisted across
  restarts).
- **Window memory** — the app remembers its size, position and maximized
  state across restarts.

### Changed

- **Cleaner headers** — token counts and costs no longer clutter the pane
  headers and title bar. They are still fully tracked: open the usage drawer
  for global/all-time stats or the per-pane stats popover for one agent.
  Pane headers now show what matters when working on a subscription: how
  much of the context window is left (`91k/200k free`) next to the context
  donut.
