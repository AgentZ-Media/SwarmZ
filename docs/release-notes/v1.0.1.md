# v1.0.1 — PR agents: Review & Review-and-merge buttons

Every pull request in the GitHub panel now carries two spawn buttons — on the
list rows and in the PR detail view:

- **Review** — spawns a fresh agent on the project whose first message is a
  full PR brief (number, title, author, branches, live checks/review/merge
  state, URL). It reviews the PR via `gh pr view`/`gh pr diff` plus the
  touched code in the repository and reports a clear verdict: blocking
  issues, non-blocking suggestions, overall assessment. It never posts to
  GitHub and never merges.
- **Review & merge** — the same review, plus a guarded merge step: only with
  no blocking findings, green checks and a mergeable PR does the agent run
  `gh pr merge` — and that command always escalates to a **human approval
  click** (it is destructive by classification), so merging stays your
  decision. Anything blocking is reported instead.

Properties of the spawned agent:

- Human-triggered and independent of the integration master toggle — it runs
  over your local `gh`, like typing the prompt yourself.
- Workspace sandbox, no worktree; the prompt forbids switching the shared
  checkout's branch or `gh pr checkout`ing in it (build/test work belongs in
  a separate worktree).
- GitHub-authored fields (title, author, branch names) are embedded with the
  autonomous-wire hygiene — single-line-flattened and JSON-string-quoted, and
  the prompt marks all PR content as untrusted data under review, never as
  instructions.

New: `prAgentPrompt` (pure, unit-tested) in `lib/github/core.ts` and
`spawnPrAgent` in `lib/github/agent.ts`.
