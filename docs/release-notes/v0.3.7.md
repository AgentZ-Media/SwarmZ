# v0.3.7 — dictation captures the first word again

Push-to-talk no longer swallows the start of what you say, and the recording
waveform actually reacts to your voice.

## 🎙️ No more clipped start

- On WKWebView the mic's WebRTC processing (echo cancellation + noise
  suppression) ramps up for ~0.6–1 s after the mic opens, feeding pure silence
  in the meantime — so the first word of a quick push-to-talk dictation was
  lost, even though the recording pill already showed.
- Dictation now opens the mic with that processing disabled, so real audio
  flows within ~150 ms. The first word is captured.

## 📊 Waveform reacts to your voice

- With the processing off the raw mic signal is much quieter, which left the
  waveform pill stuck as a flat line.
- The waveform now auto-scales to your actual level (a decaying running peak)
  instead of a fixed gain, so it deflects clearly on any microphone.
