## SwarmZ v0.2.3 — workspaces, fleet overview & command palette 🗂️

The biggest restructuring so far: SwarmZ grows from a single grid into a
multi-workspace app built for running many agents across many projects
at once.

### New

- **Workspaces** 🗂️ — tabs in the title bar, each with its own tiling
  grid. A workspace is whatever you need: one repo, one feature with
  several worktrees, or a mixed monitoring wall. The first project
  folder names the tab automatically; every tab shows a live status dot
  and a `busy/total` count. Switch with `⌘1–9` (or `⌘⇧[` / `⌘⇧]`),
  rename with a double-click, reorder by dragging, close with `⌘⇧W`
  (with a warning while agents are inside). Tabs — name, order, default
  folder — survive restarts.
- **Move agents between workspaces** ✋ — drag a pane's header onto
  another tab and the agent moves there with its terminal, scrollback
  and running processes fully intact. Under the hood terminals now live
  outside React and panes merely dock onto them.
- **Fleet overview** 🛰️ — `⌘E` zooms out to every workspace at once,
  live: real terminals scaled down, not snapshots. Workspaces that need
  attention pulse blue; click any pane to jump straight to that agent.
  The perfect second-monitor view while a dozen agents work.
- **Command palette** ⌨️ — `⌘K` fuzzy-jumps to any agent or workspace
  and reaches every global action without the mouse.
- **Attention jump** 🔔 — `⌘⇧A` cycles through the agents currently
  waiting for your input, across all workspaces.

### Changed

- The title bar's global `N working · M idle` counter is replaced by the
  per-tab live counts — the same signal, but it now tells you *where*.
- Floating terminals follow their owner's workspace; detached ones stay
  visible everywhere.
- The New Agent dialog prefers the current workspace's default folder.

### Under the hood

- Terminals (xterm + PTY) are owned by a persistent host outside the
  React tree; a PTY now ends exactly when its agent or floating terminal
  is removed — never because a pane re-rendered.
- All workspace grids stay mounted; hidden ones are only `visibility:
  hidden`, so background agents keep streaming at full speed. The WebGL
  renderer falls back to the DOM renderer if the browser ever drops a
  context with many live terminals.
