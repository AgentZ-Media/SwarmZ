# v0.3.0 — production-hardening release

Every finding of the full production audit, fixed in one release. No new
features — this is the "safe to use as a daily driver" release.

## Data safety

- **Settings/session store is crash-safe now**: `swarmz.json` gets a backup
  on every healthy launch; a corrupt/truncated file (power loss, disk-full
  mid-write) is moved aside as evidence and the last good backup is restored
  — no more silent factory reset.
- **Quit flushes everything**: tab renames, workspace order and usage
  history can no longer be lost to a fast ⌘Q (previously only the grid
  snapshot was flushed).
- **Session restore is field-tolerant**: one malformed entry in the grid
  snapshot only costs that entry, not all restored sessions.

## Worktrees

- Closing a worktree pane re-checks the worktree right before deletion — a
  "clean" verdict that aged while a confirm dialog sat open can no longer
  force-delete work that appeared in between.
- "Detach" (keep a floating terminal alive) no longer deletes the worktree
  under the detached shell.
- A hand-deleted worktree folder with unpushed commits is now shown as risky
  in the panel (two-step confirm) instead of one-click "clean up".
- Repo roots whose scan failed (unmounted volume, broken git override) are
  no longer pruned from the registry — orphaned worktrees stay discoverable.

## Guard rails

- Global shortcuts no longer fire through open dialogs — a habitual ⌘W with
  Settings open used to kill the pane *behind* the dialog.
- ⌘W on an empty workspace no longer falls through to the native "Close
  Window" menu item (which closed the whole app).
- Closing a pane while Claude is actively working now asks first.
- Floating terminals with running processes (dev servers, builds) now block
  quit like they already blocked pane close; exited panes no longer raise
  false quit warnings.

## Dictation

- Fixed a push-to-talk race where releasing ⌘ while the mic was still
  opening left the recording running (worst case: minutes of room audio
  auto-submitted to an agent).
- Hold-to-talk no longer arms while a dialog is open or while typing in a
  text field.
- A long transcription can be cancelled (mic button or the pill).
- Fixed a race that could silently drop the last dictation segment when
  stopping right at a 50s boundary.

## Local speech model

- Downloads are SHA-256-verified against a pinned HuggingFace revision — a
  captive-portal error page can no longer install as a "working" model, and
  a corrupt install self-detects and offers re-download.
- Downloads resume for real (HTTP Range) and time out on stalled
  connections instead of wedging until restart.

## Performance & stability

- PTY input goes through a per-session writer thread — pasting into a hung
  terminal (Ctrl-S, stuck TUI) can no longer freeze the whole app.
- Usage parsing, PTY kill and the local-STT status check moved off the main
  thread — no more multi-second beachball on first launch with a large
  `~/.claude/projects` history.
- Git subprocesses have hard timeouts — one repo on a dead network volume
  no longer freezes git status app-wide forever.
- A pane whose folder vanished now shows a clear error instead of an
  eternally black "starting" terminal.
- Closed shells are reaped (no more `<defunct>` zombie entries).

## Security

- A strict Content-Security-Policy (previously none).
- The OpenRouter API key is stored via the macOS Security framework — it no
  longer appears in process arguments and the keychain item is ACLed to the
  SwarmZ binary. (Reading a key saved by an older version triggers the
  standard macOS consent prompt once.)
- package.json script names are sanitized before becoming quick-command
  buttons — a malicious repo can no longer hide a second shell line behind
  an innocent-looking button label.

## Fixes

- "Reset all-time usage" works now (`window.confirm` is a no-op in WKWebView
  — replaced with an inline two-step confirm).
- Expired/revoked Claude logins hide the subscription meters instead of
  showing stale numbers forever.
- Subscription-meter, profile-delete, preset-name, usage-drawer (Escape +
  focus), pane-stats-in-⋯-menu and tab-tooltip paper cuts.
- The New Agent dialog no longer wipes typed input when a background
  settings write lands, can no longer silently downgrade a requested
  worktree to a plain pane, and "Cancel" during worktree creation actually
  cancels.
- Claude/git binary path overrides are validated with inline feedback and
  only apply on blur instead of per keystroke.
