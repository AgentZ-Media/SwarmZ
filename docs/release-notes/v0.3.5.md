# v0.3.5 — title bar is properly draggable again

A focused fix for window dragging that the previous attempt didn't fully
resolve.

## 🖱️ The whole title bar drags now

- v0.3.4 tried to restore title-bar dragging, but the window could still only
  be moved from a few millimetre-wide gaps between the toolbar elements.
- Root cause: the title bar used a bare `data-tauri-drag-region`, which only
  drags when you click the `<header>` element *itself* — i.e. the tiny flex
  gaps. The `-webkit-app-region` CSS that was meant to cover the rest is a
  Chromium/Electron feature that macOS WKWebView ignores entirely, so it never
  did anything.
- Switched to `data-tauri-drag-region="deep"`: the entire title-bar area is now
  draggable, while Tauri keeps real buttons, tabs and inputs clickable. You can
  grab the window from anywhere along the bar, including the empty space in the
  workspace-tab strip.
