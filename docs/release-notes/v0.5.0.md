# v0.5.0 — Vibe Mode, Maestro & a smarter Orchestrator

A second way to work: native Codex sessions with a UI of their own, an
orchestrator that now has a name and a memory, and a chat that finally reads
like a chat.

## Vibe Mode (⌘⇧V)

- An app-wide second view where Codex agents run as native `codex app-server`
  sessions instead of terminals — rendered as a real chat, not a scrollback.
  The grid world stays mounted and untouched; the mode switch (title bar or
  ⌘⇧V) flips between them.
- **Focus + Rail layout**: a session rail on the left (signal-triad status,
  model · effort · context per card) and a focus stage on the right —
  Orchestrator-first, so the Conductor (the orchestrator chat) owns the stage
  by default and `@session` in the composer talks to a session directly.
- **Structured transcript**: agent messages, commands (with exit codes and
  live output), file changes as real diffs, and plan/todo progress — each as
  its own card, virtualized so long threads and many parallel sessions stay
  fast.
- **Diff cards** via `@git-diff-view` with off-main-thread syntax highlighting,
  collapse-by-default for big diffs and a per-turn diff panel.
- **Approvals you can actually use**: a workspace-write session's approval
  requests become an amber composer takeover (approve / allow for session /
  decline / cancel, ⏎/⎋, a `1 of N` queue) with a cross-session banner so
  nothing waits unseen. Per-session full-access toggle, applied on the next
  turn.
- The orchestrator sees and drives sessions too: `fleet_snapshot` lists them
  with exact status, `create_panes native:true` spawns them, and pings fire
  when one finishes.

## Maestro — the orchestrator persona & memory

- The orchestrator now has an editable persona (**Maestro** 🎼 by default;
  Hive and a neutral preset in Settings → Orchestrator) compiled into a fixed
  operating core — personality sets the voice, never the tools or safety
  rules.
- A small, curated memory: a `remember` tool writes durable facts to an
  editable file (hard-capped), injected into every new chat. It proposes what
  to note and only writes on your go-ahead.

## A chat that reads like a chat

- Reading-width column and larger, calmer typography across both the
  orchestrator panel and Vibe.
- Tool calls fold into one quiet activity stream — human verbs ("Checked the
  fleet", "Read «pane»"), single steps shown inline, raw tool names only in a
  tooltip. Status pings and warnings share the same line language.
- File paths render as compact pills (filename + optional `:line`, full path
  in the tooltip), never as links.
- Model and reasoning effort are visible everywhere and changeable on the fly
  — the picker's catalog is the real `model/list` from your account, with a
  per-provider default in Settings that new chats inherit.

## Layout-aware orchestration (grid mode)

- `fleet_snapshot` now reports each workspace's real pixel size and tiling
  tree, and `create_panes` takes placement intent (`workspace` /
  `arrangement` / `beside`). Guardrails keep panes readable: too many for the
  space overflow into a fresh workspace automatically, and same-project panes
  stay together — no more "each pane smaller than the last".

## Under the hood

- The `codex app-server` integration is now a single shared host
  (`codex/host.rs`) that both the orchestrator and Vibe sessions build on —
  one place for spawning, framing, resume and respawn.
- A frontend unit-test suite (`pnpm test`, vitest) covers the new pure logic.
