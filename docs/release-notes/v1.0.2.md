# v1.0.2 — Conductor mission upgrades

Four upgrades that let the Conductor run LARGE missions (dozens of tasks,
parallel agent waves) robustly and largely autonomously.

## Fleet-event coalescing — one budget unit per wave

Agent finishes that pile up while an autonomous turn is running no longer
burn the autonomy budget one agent at a time: they are batched into a single
`[fleet events]` turn (up to 8 events, the excess follows as the next turn).
A 15-agent wave used to trip the circuit breaker at the 6th finish — now it
costs one budget unit per wave. The breaker itself is unchanged: same caps,
same latch, still only a human message re-arms it, and the injection
hardening stays intact (only the batch header carries an event marker; the
per-event sections are marker-stripped, shared contract lines deduped).

## The Conductor can explore the project itself

Two new read-only tools, `list_files` and `read_file` (31 → 33 tools), give
the Conductor a bounded window into the project tree — for grounding its
decomposition and agent briefs in the real repo layout before spawning
anyone. Strictly confined: relative paths only, no-follow walks, hidden
files/folders (`.env`, `.git`, `.swarmz`, …) are never served, listings and
reads are capped, binary files refuse. Doctrine: orientation, never a scout
replacement.

## Missions: a durable task board

Large goals now run against ONE task-board plan under `.swarmz/plans/` as
the mission's durable state — checkbox lines per task, updated after every
finish or decision. After any interruption (tripped breaker, restart, missed
event) the Conductor re-orients from the board plus the live fleet state
instead of its memory: read the board, reconcile, continue from the open
items.

## Agents know their worktree

Conductor-spawned agents placed in a worktree receive an automatic workspace
briefing ahead of their first task: the worktree path and branch, where the
main repository lives ("leave it untouched"), that dependency dirs like
`node_modules` were not copied (install first when the task needs builds,
tests or a dev server), and the one-writer rule for shared lanes.

Also in this release: CodeRabbit review hardening for the new explore
surface (Windows drive-prefix paths refused fail-closed, bounded directory
enumeration, strengthened injection tests).
