# v1.0.0 — The Codex-only Swarm Manager

SwarmZ 1.0 is a ground-up rebuild. The old terminal/PTY grid, the Claude
runtime, the OpenRouter brain and voice dictation are gone; what remains is a
single, coherent product — a native macOS **swarm manager** over the
`codex app-server`, with an **autonomous AI Conductor per project** on top.

The major version marks the rebuild: nothing about the data model, the runtime
or the UI carries over from the 0.x terminal-grid app (which lives on in git
history and the `main` branch). This is a new application under the same name.

## What SwarmZ is now

Every agent is a **native Codex session** driven straight through the Codex
app-server — no terminal emulation. Work arrives as structured cards: streaming
assistant messages (rendered as markdown), collapsible command output,
syntax-highlighted diff cards, plans, and approval prompts you allow or decline
right in the composer.

Sessions are organized in **project tabs** — one per folder, deduped by
canonical path, resumable across restarts. Each project keeps its own swarm,
its own Conductor, and its own memory.

## The Conductor — an autonomous engineering lead, per project

Each project tab gets its **own Conductor**: its own `codex app-server` process
(spawned lazily, reaped after 15 idle minutes, transparently resumed), its own
chats, its own scoped memory, and a project-scoped tool arsenal. It is not a
chatbot bolted on the side — it is a team lead:

- **It delegates by default.** Give it a goal and it decomposes the work onto
  1–8 named sub-agents, each in its own git worktree, and drives them:
  `prompt_agent` steers a busy agent mid-turn, `review_agent` runs a detached
  Codex review of finished work, `set_agent_config` retunes model/effort/access
  per task.
- **It manages worktrees** itself — creates them per lane, re-homes agents,
  and cleans up under a safe gate that never force-removes work.
- **It sets timers** for its own follow-ups and writes **plan documents** into
  a confined `.swarmz/plans/` area (its one sanctioned write surface — it never
  edits your code).
- **It is genuinely autonomous.** Fleet events — an agent finishing, an agent
  asking for direction, a routine approval, a timer firing, a long-idle lane,
  a PR changing — wake it for a budget-gated turn where it judges the result,
  hands out follow-ups, reports back, and learns your preferences. Agents
  deliver schema-forced status reports so the Conductor always knows what
  happened.
- **Autonomy is hard-capped.** A per-project circuit breaker (5 consecutive /
  20 per hour) latches on exhaustion and only a human message re-arms it — the
  cap against runaway or prompt-injected cascades. Every event payload is
  treated as untrusted data, never as instructions.

## Approvals stay human — and safe by construction

Session approvals surface as cards in the composer; the human always holds final
authority. The Conductor gets a *fast lane* for genuinely routine approvals
(read-only/test commands, pure in-tree edits) — but the classification is
**fail-closed and Rust-anchored**: anything with shell metasyntax, interpreters,
foreign paths, deletes or renames is destructive and stays human-only, enforced
server-side next to the blocked responder. In doubt, always the human.

## GitHub, opt-in and local-only

SwarmZ reads and (optionally) manages a project's GitHub context over your
**locally installed `gh` CLI** — there is **no OAuth, no login flow, and no
token ever touches SwarmZ**. Read-only detection and the PR panel work always,
degrading gracefully when gh is missing or logged out. A single Settings master
toggle (default off) arms the Conductor's PR tools, the write gate, PR-approval
routing and a PR watcher that can wake the Conductor on changes. There is
deliberately **no merge or close** command anywhere — merging stays a human
action; pushes are plain (never forced) and never touch the default branch.

## Vibe v3 — the interface

- A **fleet grid** of live agent cards for the active project — status, worktree
  branch, ±diff, a mini feed and quick approvals, all on one dot-grid canvas —
  and a **focus stage** to expand one agent full-window.
- A persistent, resizable **Conductor sidebar** (⌘B) with a project-scoped
  `@agent` composer.
- A slim **Deck** status bar: global fleet counters, a needs-you triage queue
  (⌘⇧A), an event ticker, account-level Codex plan meters, a PR indicator and
  the Conductor status dot.
- Diffs render through **@pierre/diffs** with a SwarmZ Shiki theme
  (WKWebView-safe, no Oniguruma WASM).
- **Context compaction**: sessions and Conductor chats show a context gauge
  with one-click compaction, and a session near its model window auto-compacts
  before the next turn (Settings toggle, on by default) — the visible
  transcript always stays intact, only the model-side context shrinks.
- One accent, one signal language (active / needs-you / finished /
  failure), a full design-token system, and a `prefers-reduced-motion`-aware
  motion doctrine. Dark only, by design. See `DESIGN.md`.

## Privacy

Your app state — sessions, chats, notes, settings, memory — stays on your
machine. Model traffic goes to OpenAI through the Codex CLI (your ChatGPT
sign-in); GitHub actions go through your local `gh`. SwarmZ itself holds no
account, no token and no server.

## Requirements

- macOS (Apple Silicon)
- [Codex CLI](https://developers.openai.com/codex/cli) ≥ 0.144, signed in with
  your ChatGPT account
- Optional: the [GitHub CLI](https://cli.github.com) (`gh`) for the GitHub
  integration
