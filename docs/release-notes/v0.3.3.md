# v0.3.3 — usage tracking for manually resumed sessions

A small fix release for per-pane usage tracking.

## 📊 Resumed sessions are detected again

- **Manual `claude --resume` in a fresh pane now shows usage**: resuming an
  old session appends to its pre-existing session file, which the previous
  discovery (only files *created* after the pane spawned) could never match —
  the pane stayed stuck on "no activity" with no context donut, no matter how
  much Claude did.
- Discovery now falls back to the session file most recently **modified**
  after the pane went busy, so the resumed session is picked up with its
  first new turn: the context donut shows the real occupancy immediately,
  and stats count from the resume onward (the full history is included once
  the session is latched, as before).
- SwarmZ's own workspace restore (`--resume` injected at spawn) was never
  affected — its session id is already known.
