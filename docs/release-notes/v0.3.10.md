# v0.3.10 — Worktree cleanup

- Added a **Clean up safe** action to the worktree panel for removing clean,
  unattached SwarmZ worktrees in bulk.
- Added a **Finish worktree** action to worktree pane menus, using the same
  safe close flow as the pane close button.
- Workspace close now offers **Close & clean safe**, which stops the workspace
  panes and removes only worktrees that still re-check clean with no local-only
  commits.
- Worktree rows now describe unreachable commits as **local-only**, making the
  cleanup safety check clearer for pushed branches and pull request workflows.
- Added product/design context files for future UI work and kept worktree
  cleanup docs in sync.
