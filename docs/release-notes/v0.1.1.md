## SwarmZ v0.1.1 — context gauge, live pricing & a big perf pass ⚡

### New

- **Context gauge** 🍩 — a small donut in every pane header shows how full
  the agent's context window currently is (blue → amber → red as it fills).
  Hover for exact numbers, e.g. `Context: 73.4k / 1m (7%)`. 1M-context
  sessions are detected automatically.
- **Live pricing** — per-model costs (incl. cache write/read) are now fetched
  from the free OpenRouter model catalog and refreshed daily, so new models
  like Fable are priced correctly. While offline, a hardcoded per-family
  fallback applies.

### Performance

A full energy-consumption pass — SwarmZ now sits quietly in the background:

- **Incremental usage parsing** — session JSONLs are no longer re-read and
  re-parsed in full every few seconds. Only newly appended bytes are parsed;
  unchanged files cost a single `stat`.
- **Smarter file-watcher** — changes from Claude sessions you're *not*
  displaying (other projects, other terminals) no longer trigger refreshes,
  and all refresh triggers are coalesced (min. 2s apart).
- **Coalesced terminal output** — PTY output is batched (≤12 ms) into single
  events addressed per agent, instead of waking every pane for every 8 KiB
  chunk a TUI redraw produces.
- **Idle hygiene** — only the active pane's cursor blinks, and usage polling
  pauses entirely while the window is hidden.
