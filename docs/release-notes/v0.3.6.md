# v0.3.6 — correct 1M context window detection

The context donut now recognises the 1M window from the very first turn,
instead of getting stuck at 200k until a session actually filled past it.

## 🧠 1M context is detected immediately

- Sessions on a 1M-context model (e.g. Opus 4.8) showed only **200k** in the
  context donut and jumped to 1M only once usage crossed 200k — because the
  session log records the **bare** model id (`claude-opus-4-8`); the `[1m]`
  marker is stripped from the response, so there was no below-200k signal.
- SwarmZ now reads which models you've actually run at 1M from Claude Code's
  own config (`~/.claude.json` → per-project `lastModelUsage`, where the real
  `…[1m]` variant is recorded) and uses that to size the window. The donut
  shows the correct 1M occupancy from the first turn, in every pane.
- The detection is cached against the config file's modification time, so it
  costs nothing on the usual usage-refresh polls.
