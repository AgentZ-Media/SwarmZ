## SwarmZ v0.2.4 — session restore 🔁

Quit SwarmZ and the next launch brings everything back: every workspace,
the exact pane layout, and each Claude pane reopens its previous
conversation.

### New

- **Session restore** 🔁 — the grid is saved continuously, so a restart
  (or a crash) restores all workspaces with their exact pane layout.
  Every pane that was running Claude resumes its conversation via
  `claude --resume` — context, history and the usage stats latch right
  back on. Panes running other commands (dev servers, shells) restart
  with their original startup command.
- **Settings toggle** ⚙️ — "Restore agents on launch" in the Terminal
  section, on by default. Turn it off to always start with a clean
  slate.

### Notes

- Resume restores the *conversation*, not in-flight work: an agent that
  was mid-task when you quit comes back at the prompt with its full
  history (the quit guard still warns you about busy agents first).
- Floating terminals are plain shells without a session — they don't
  come back.

### Under the hood

- New persisted `grid` snapshot (agents incl. their latched claude
  session id + the tiling trees), debounced on every grid mutation and
  flushed on all quit paths before the window closes.
- The `--resume` flag is injected only at PTY spawn — startup commands
  stay clean for split-prefill and persistence; compound commands and
  explicit `--resume`/`--continue` are left untouched.
