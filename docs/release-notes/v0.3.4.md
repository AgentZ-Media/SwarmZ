# v0.3.4 — subagent usage tracking & title-bar drag fix

Per-agent visibility into the subagents your sessions spawn, plus a fix for a
title bar that had become hard to drag.

## 🤖 Subagents are now tracked per agent

- When a Claude session spawns subagents (the Task tool), each one runs in its
  **own context window** and was previously invisible per-agent — only the
  global usage dashboard counted it, blended into the parent.
- SwarmZ now reads each subagent's own session file
  (`<project>/<session>/subagents/`) and shows them as a thin strip of chips
  **under the pane header**: agent type, a mini context bar with its own
  occupancy %, and a live "running" indicator. Hover any chip for the full
  breakdown (model, context, tokens, cost).
- The per-agent stats popover gained a matching **Subagents** section.
- Kept deliberately separate from the parent's context donut — subagents have
  their own windows, so summing them would be misleading.

## 🖱️ Title bar is draggable again

- After the workspace/tab strip landed, the tab container spanned the whole
  middle of the title bar while opting the entire region out of window
  dragging — so the window could only be moved from a few narrow spots.
- Only the tabs and the "+" button opt out now; the strip's empty space is
  draggable again, restoring normal title-bar behaviour.
