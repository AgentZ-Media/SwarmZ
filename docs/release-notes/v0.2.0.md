## SwarmZ v0.2.0 — drag & drop panes, settings window & git at a glance 🧲

### New

- **Drag & drop pane rearranging** 🧲 — grab a pane's header and drop it on
  another pane: edges dock it to that side (left/right/top/bottom), the
  center swaps the two panes. Divider resizing and pane drags both show a
  translucent preview while dragging and apply on release, so terminals
  never reflow mid-drag — `Esc` cancels either gesture. Panes never
  remount, so scrollback survives every rearrangement.
- **Settings window** ⚙️ — `⌘,` (or the gear in the title bar) opens
  Settings: default terminal font size, default startup command for new
  agents, path overrides for the `claude` and `git` binaries, update
  controls (manual check + auto-download toggle) and an About panel.
- **Git at a glance** 🌿 — panes whose folder is inside a git repo show the
  branch plus live diff counters: `+added` / `−removed` lines and untracked
  files, refreshed every few seconds and strictly read-only. Repo name and
  an **Open repo in browser** action live in the pane menu and stats
  popover.
- **Responsive pane headers** 📐 — narrow panes shed secondary info (folder
  path, git counters, gauge readout, split buttons) step by step until only
  the title, model badge and context donut remain; everything stays
  reachable via tooltips and the ⋯ menu.

### Improved

- **Plan limits** are now resilient to network hiccups: a failed fetch
  keeps the last known values instead of hiding the meters — they only
  disappear when there is genuinely no Claude Code login.
- With several agents working in the same folder, panes no longer risk
  latching onto a sibling's Claude session for usage tracking.
- The title bar shows the app icon, and the manual update check moved into
  Settings.

### Under the hood

- New read-only git status backends (`src-tauri/src/git.rs` native,
  `/api/git` in web mode) polled once per distinct folder, paused while the
  window is hidden.
