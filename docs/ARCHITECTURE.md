# SwarmZ architecture notes

Deep-dive companion to `CLAUDE.md` ‚Äî the full how-and-why per subsystem.
Update this file when behavior described here changes.

## Transport & backends

`src/lib/transport.ts` picks the backend at runtime (Tauri vs the abandoned web fallback). Backend capabilities are declared in `backend-types.ts` and implemented in `backend-tauri.ts`. Native-only features may skip the backend interface and use `@tauri-apps/api` directly (e.g. `lib/dnd.ts`, `lib/openrouter.ts`, `lib/worktree.ts`).

## Store & persistence

`src/store.ts` holds all app state. Persisted: profiles (including agent runtime: `claude` / `codex` / `shell`), usage history, workspace tabs (`Workspace`: name/order/defaultCwd + active tab, store key `workspaces`, debounced via `schedulePersistWorkspaces`), the grid snapshot for session restore (store key `grid`, see below) + app settings (`AppSettings`: last used folder, default font size, default startup command, claude/codex/git path overrides, auto-update, restore-on-launch, voice-dictation prefs (engine cloud/local, hotkey mode, auto-submit, cleanup toggle/model/prompt, STT model, mic device id+label), worktree repo registry (`worktreeRepos`, written by the worktree feature, not the dialog) ‚Äî edited in `SettingsDialog.tsx`, opened via the title-bar gear or `‚åò,`) (Tauri store `swarmz.json` / localStorage). Always mutate settings through `updateSettings` (merges + persists the whole object). In-memory only: floating terminals, focus mode, fleet/palette state. Window size/position persists natively via `tauri-plugin-window-state`.

New agents and splits go through the New Agent dialog and land in the active workspace; splits prefill it from the source pane (runtime, cwd, profile, startup), plain "new" prefills the configured default startup command (picking a profile overrides it) and cwd from workspace defaultCwd ‚Üí profile default cwd ‚Üí last used folder. The dialog resets its fields only on the opening edge (background store writes while it is open must never wipe typed input). Runtime path overrides are applied at PTY spawn (`applyRuntimePath` in `utils.ts` ‚Äî replaces a leading `claude` or `codex` token only); binary-path overrides commit on blur and are stat'ed via `path_is_file` with inline feedback.

## Session restore

The live grid is snapshotted continuously (store key `grid`, debounced `schedulePersistGrid` in `store.ts`: each agent's restore-relevant fields incl. runtime and the latched `sessionId`, plus the per-workspace layout trees and active panes) and flushed on every quit path together with the workspaces + usage-history debounces (`flushAllPersists`, called in `lib/quit.ts` before the window actually closes). On launch `hydrate()` rebuilds panes from the snapshot only if `settings.restoreAgents === true` (default **off**) and no agent was already spawned; torn snapshots degrade gracefully **per entry** (snapshot carries `version: 1`; agents missing id/workspaceId are skipped, other fields get safe defaults, a corrupt tree only costs its own workspace, panes without their agent are pruned, agents without a pane are dropped, fully-pruned trees normalize to `null`).

The store file itself has a crash net: `storefile.rs` runs in Tauri `setup` before the webview loads ‚Äî a healthy `swarmz.json` refreshes `swarmz.json.bak`, a corrupt one (torn write, disk-full) is moved aside as `swarmz.json.corrupt-<ts>` and the backup restored, so the plugin's silent treat-unreadable-as-empty behavior can no longer factory-reset the app.

Restored agents carry the transient `Agent.resume = sessionId` ‚Äî applied only at PTY spawn (`Terminal.tsx` ‚Üí `resumeRuntimeStartup` in `utils.ts`, using `claude --resume <id>` for Claude and `codex resume <id>` for Codex; compound commands and explicit resume forms pass through), so `agent.startup` stays clean for split-prefill and re-persisting. Claude resume keeps the same session id, so usage latches right back onto the same jsonl ‚Äî and a latched session is parsed without the since-filter (`usage_for_session` in `usage.rs`), so stats and the context donut include the pre-restart turns despite the reset `createdAt`. Claude session latching is gated on real activity: an unlatched pane only runs session discovery once Claude first reported busy (`Agent.firstBusyAt`, stamped in `setActivity` from OSC 9;4), and discovery then first matches files born around that first activity (App.tsx usage refresh). Codex latching uses `~/.codex/sessions/**/*.jsonl` `session_meta.cwd` / `session_id` metadata and starts at pane creation because Codex does not rely on Claude's OSC 9;4 signal; already-latched sibling ids are excluded. Floating terminals are not captured.

## Workspaces

Workspaces are the top-level containers (tabs in the title bar, `WorkspaceTabs` in `TitleBar.tsx`): each has its own tiling tree (`layouts`/`activePaneIds` keyed by workspace id). Deliberately not bound to a project ‚Äî `defaultCwd` only prefills the New Agent dialog; the first agent with a cwd auto-names the tab (`folderName`, until the user renames it) and sets `defaultCwd`. Tabs show an aggregated status dot + `busy/total` count, rename on double-click, reorder by drag, close via hover-X/middle-click (`CloseWorkspaceDialog.tsx` confirms when agents are inside; closing the last workspace spawns a fresh one ‚Äî there is always ‚â•1). Shortcuts: ‚åò1‚Äì9 switch, ‚åò‚áß[/‚åò‚áß] cycle, ‚åò‚áßN new, ‚åò‚áßW close. Dragging a pane header onto a tab moves the agent there (`moveAgentToWorkspace`; tab highlight via `tabDropTarget`).

## Workspace presets

Store key `workspacePresets`, helpers in `lib/presets.ts`: named layout trees whose leaves are pane templates (`PresetPaneNode`: cwd unset = *inherit* the folder asked once at load via `LoadPresetDialog.tsx` ‚Äî skipped when every pane is pinned; startup unset = default startup command, `""` = plain shell; plus name/profileId/color). Seeded with Solo/1√ó2/2√ó2/1+2 only when the key is **missing** ‚Äî an emptied list stays empty. Loaded from the empty-workspace screen (`EmptyState` in `WorkspaceLayer.tsx`, cards with `PresetThumbnail.tsx` mini-grids; ‚åò‚áßN ‚Üí empty tab ‚Üí cards covers the new-workspace flow): `applyPreset` builds all agents + the live layout in one pass, fills the active workspace if still empty (else a new tab) and names the tab after the preset (`renamed: true`). Created via ‚åòK ‚Üí "Save workspace as preset" (`SavePresetDialog.tsx` ‚Üí `presetLayoutFromGrid` snapshots the live grid; agent names only carry over when user-renamed). Managed in Settings ‚Üí Presets (rename, per-pane folder/command, remove panes).

## Terminals & PTYs

**Terminals live outside React** in `src/lib/term-host.ts` (xterm + PTY per id; `TerminalView` only attaches/detaches the host element). Unmounting a pane detaches the DOM but keeps the terminal running ‚Äî that is what lets panes remount across workspace grids without losing scrollback/PTY. PTYs die exactly on store removal: `removeAgent` / `removeFloatingTerminal` ‚Üí `destroyTerm` (NOT on unmount). The WebGL addon disposes itself on context loss (browser caps live WebGL contexts; with every workspace mounted that's reachable) and xterm falls back to the DOM renderer. A failed `pty_spawn` (deleted cwd, removed worktree) writes an error banner into the terminal and flips the pane to `exited` instead of hanging on "starting".

PTY input goes through a per-session writer thread in `pty.rs` (channel send, single consumer): `pty_write` stays a fast sync command (keystroke ordering preserved) and can never block the main thread on a PTY whose foreground process stopped reading. `pty_kill` is async (portable-pty's kill sleeps up to ~250 ms in its SIGHUP grace loop) and sessions are killed outside the manager mutex + reaped on a detached thread (no `<defunct>` zombies).

All workspace grids stay mounted in `WorkspaceLayer.tsx`; inactive ones are `visibility:hidden` + `pointer-events-none` (never unmounted, never resized ‚Äî hidden agents keep working). **Fleet overview** (‚åòE / grid button): the same wrappers are scaled into a card grid with pure CSS transforms (no size change ‚Üí no PTY resize, terminals stay live), with `FleetCard` chrome on top ‚Äî name + status pill, blue pulse on attention, and click-through that maps the cursor onto the exact pane (`paneRects`) and jumps to it. Escape/‚åòE exit (entering blurs the focused terminal so Escape arrives); exit animates via a render-phase `fleetClosing` flag, mirroring the focus-mode pattern.

## Command palette & custom commands

**Command palette** (‚åòK, `CommandPalette.tsx`, cmdk): fuzzy-jump to any agent/workspace + all global actions; custom commands surface as their own group **only while a search is typed** (folder commands of the target pane + globals) and run with the same paste/‚åò‚Üµ semantics as ‚åò‚áßK. **‚åò‚áßA** (`attentionJump`) cycles through agents that need input (bell attention or OSC "waiting") across all workspaces.

**Custom commands** (‚åò‚áßK, `InsertCommandPalette.tsx`): user-saved prompt snippets, global + per folder (store key `customCommands` ‚Üí `CustomCommandsData`, folder lists keyed via `presetKey` ‚Äî deliberately separate from the floating-terminal `commandPresets`, which are executed shell commands). Selecting **pastes** into the target pane via `term.paste()` without submitting; ‚åòEnter submits via a **separate** `ptyWrite "\r"` after the paste (a `\r` inside the bracketed paste is only a literal newline in claude's input box) ‚Äî the paste/submit core is shared with ‚åòK in `lib/insert-command.ts` (`insertCommandText`); a `{{input}}` command picked in ‚åòK detours through the picker's form via store `commandPickerPreselect` (in-memory, consumed on open). Target = focused agent, else active pane; agent panes only (floating terminals have no focus tracking in the store ‚Äî extending later is just target resolution, `insert()` only needs a pty id). Placeholders live in `lib/command-vars.ts`: `{{folder}}/{{cwd}}/{{branch}}/{{agent}}` fill from the pane, `{{input:Label}}` asks inline before inserting (same label twice = same value), unknown `{{‚Ä¶}}` stay literal. Managed in Settings ‚Üí Commands, quick-add directly in the picker. The ‚åò‚áßK keydown branch in `App.tsx` must stay **before** the plain ‚åòK branch (which doesn't check shift). While any dialog is open, global shortcuts early-return (only ‚åòK/‚åò‚áßK may still close their own palettes) ‚Äî keydown bubbles through Radix's focus trap, and an unguarded ‚åòW would kill the pane behind the dialog; ‚åòW is also always `preventDefault`ed so an empty workspace can't fall through to the native Close Window menu item.

## Quick notes

**Quick notes** (‚åòN, title-bar note button, `QuickNotesPanel.tsx`): a right-side slide-in drawer (same overlay/z-40/`animate-slide-in-right` pattern as the usage drawer) holding lightweight checklists ‚Äî one **global** list plus one **per project folder** (store key `quickNotes` ‚Üí `QuickNotesData`, folder lists keyed by repo root). The drawer opens in the scope of the active pane via `activeProjectRoot()` in the store ‚Äî a worktree pane resolves to its **main repo**, never the `.worktrees/<slug>` path; fallback is the workspace `defaultCwd`, else Global ‚Äî and shows a chip per scope (Global, the active project, plus every folder that already has notes). Items (`NoteItem`) check off, edit inline (click the text; committing empty deletes), reorder by dragging the grip (sibling-midpoint pattern like the workspace tab strip) and can toggle to plain text lines without a checkbox (`plain`); "Clear done" drops every checked item, emptied folder lists drop their key. Every mutation persists immediately (no debounce ‚Äî nothing to flush at quit). The drawer is a `role="dialog"` so the global-shortcut guard kicks in; ‚åòN closing it is special-cased in the dialog branch of the `App.tsx` keydown handler (like ‚åòK/‚åò‚áßK), and the new-note input autofocuses on open so capture is type-Enter-done.

## Voice dictation

Hold mode = plain ‚åò push-to-talk, toggle mode = ‚åò‚áßM, per `dictationHotkeyMode`; or the mic button in pane/floating headers. Hold mode arms on Meta keydown and only opens the mic after ~250 ms held **alone** (`armHoldDictation` ‚Äî ‚åò-shortcuts never trigger it; any other keydown cancels silently via `cancelHoldDictation`; arming is skipped while a dialog is open or a real text field ‚Äî not xterm's helper textarea ‚Äî has focus), Meta keyup or window blur finishes (`finishHoldDictation`). A `holdGen` generation counter closes the getUserMedia race: if ‚åò was released while the mic was still opening, the recording is aborted instead of running hot. Recordings under ~1 s are discarded instead of transcribed (OpenRouter rejects near-empty audio). Recording happens in the webview (`lib/dictation.ts`, outside React like term-host ‚Äî getUserMedia + MediaRecorder, rotated into ~50 s segments because the OpenRouter transcription API wants ‚â§60 s uploads, 5 min hard cap; `settings.dictationMicId` picks the input device as an **ideal** deviceId constraint, so an unplugged saved mic falls back to the system default instead of failing ‚Äî chosen in Settings ‚Üí Voice via `listMicrophones()`, which enumerates lazily on picker open only (briefly opening the mic if WebKit still hides labels) and persists id + label so the selection renders without re-probing; `stopDictation` awaits an in-flight rotation so the boundary segment is never lost; the AnalyserNode feeds the waveform pill in `Dictation.tsx`, rendered inside the target's terminal container). getUserMedia opens the mic with **all WebRTC processing disabled** (`echoCancellation`/`noiseSuppression`/`autoGainControl` all `false`): in WKWebView the APM ramps up for ~0.6‚Äì1 s after the track opens and delivers pure silence ‚Äî push-to-talk was swallowing the first word; raw capture gets real samples within ~150 ms. The trade-off is a ~10‚Äì30√ó quieter signal (the APM is also what made it loud), so the waveform pill normalizes against a slowly-decaying running peak instead of a fixed gain ‚Äî don't reintroduce processing or a constant multiplier. An in-flight transcription can be cancelled (`cancelTranscription` ‚Äî mic button or the pill's Cancel).

Two engines (`settings.dictationEngine`, default `"openrouter"`): cloud transcription, optional LLM cleanup and the model catalog run natively in `src-tauri/src/openrouter.rs`; the **local engine** (`"local"`, `src-tauri/src/localstt.rs` + `lib/local-stt.ts`) runs NVIDIA Parakeet TDT 0.6b v3 int8 on-device via transcribe-rs/ONNX Runtime (statically linked at build time through ort's `download-binaries` ‚Äî no Python or system deps ever) ‚Äî the ~670 MB model (4 files from HF `istupakov/parakeet-tdt-0.6b-v3-onnx`, **pinned to a commit revision with per-file SHA-256s** in `MODEL_FILES` ‚Äî size + hash are verified before the `.part`‚Üífinal rename, and `is_installed` is size-aware so a corrupt install self-detects) is downloaded once from Settings ‚Üí Voice into `~/Library/Application Support/SwarmZ/models/` (`.part` + rename, resumable via HTTP Range, 15 s connect/30 s read timeouts so a stalled connection errors instead of wedging, progress via `localstt://progress`), loaded lazily on first dictation (~1.5‚Äì2 GB RAM, cached until unload ‚Äî switching the engine back to cloud unloads it; the loaded flag is an `AtomicBool` so `local_stt_status` never blocks on the inference mutex) and fed 16 kHz mono PCM that the webview decodes/resamples from the recording via OfflineAudioContext (`blobToPcm16k`).

Mic gating is engine-aware (`selectDictationReady`: local ‚Üí store `localSttStatus.installed`, cloud ‚Üí `openrouterStatus`); the cleanup pass always goes through OpenRouter, so with the local engine it only runs when a usable key is also stored. The API key lives in the macOS Keychain (service `SwarmZ-OpenRouter`, via the Security framework ‚Äî never in a subprocess argv; only limits.rs still uses the `security` CLI because it reads *Claude's* item) and never reaches JS ‚Äî `lib/openrouter.ts` is native-only direct `invoke` (skips the backend interface, like dnd.ts). The transcript is **pasted** like the insert picker (`term.paste`, separate `\r` only with `dictationAutoSubmit`). Cleanup uses structured output (`json_schema {cleaned}` + `require_parameters`) with `reasoning: {effort: "minimal"}` ‚Äî NOT `enabled: false`, Gemini 400s on that ("Reasoning is mandatory ‚Ä¶ cannot be disabled"); models that reject "minimal" get one retry without the reasoning field (only on a reasoning/parameter rejection ‚Äî never on timeouts/429/5xx) ‚Äî and a user-editable prompt whose default forbids translating ‚Äî dictation must stay language-agnostic. All mic UI is gated on store `openrouterStatus` (fetched at launch and on key save; `valid: null` = present-but-unverified counts as usable, only a 401/403 hides it). One dictation at a time (store `dictation`, in-memory). Mic permission comes from `src-tauri/Info.plist` (`NSMicrophoneUsageDescription`).

## Usage & limits

Claude usage is parsed from `~/.claude/projects/*.jsonl` by `src-tauri/src/usage.rs`. Parsing is **incremental**: a per-(file, since) offset cache only reads appended bytes (entries of deleted files are evicted during `usage_totals` walks); costs/by_model are recomputed from cached counters on every read (so pricing updates need no cache invalidation). Codex usage is parsed from `~/.codex/sessions/**/*.jsonl` by `src-tauri/src/codex_usage.rs`, using `session_meta`, `turn_context`, `event_msg/token_count`, `task_started` and `task_complete` rollout events for session id, cwd, model, tokens, context, limits and activity. The usage commands are async + `spawn_blocking` ‚Äî the first call after launch parses the whole backlog and must not freeze the UI. Claude pricing is fetched live from the OpenRouter model catalog (daily refresh) with a hardcoded per-family fallback for offline; Codex ChatGPT-plan rollouts do not include reliable USD cost, so Codex cost stays zero unless a future parser can estimate it honestly.

**Context-window (donut) inference** (`context_limit` in `usage.rs`): the JSONL records only the **bare** model id (`claude-opus-4-8`) even on 1M sessions ‚Äî the `[1m]` suffix is stripped from the response `message.model` ‚Äî so the 200k-vs-1M window is inferred, in order: explicit `[1m]` in the id ‚Üí `~/.claude/settings.json` `model` being this model's `[1m]` variant ‚Üí the model having actually been run at 1M before (any `projects.*.lastModelUsage` key with a `[1m]` suffix in `~/.claude.json`, harvested by `onem_models()`, cached against that file's mtime) ‚Üí observed context > 200k. The `~/.claude.json` lookup is what makes the donut show 1M from the **first** turn instead of only after a session crosses 200k ‚Äî `settings.json` is usually empty because the model is picked via `/model`, which doesn't write it there.

**Subagents (Task tool).** Claude Code writes each spawned subagent to its own jsonl in a sibling dir of the parent session: `<project>/<session_id>/subagents/agent-<id>.jsonl` (same `sessionId`, every assistant line `isSidechain:true`, plus `attributionAgent` = the agent type, e.g. "Explore"). The aggregate `usage_totals` walk (`WalkDir`, recursive) already counts these ‚Äî blended into the parent session. For **per-agent** visibility, `usage_for_session` additionally scans that subagents dir (`subagents_for_session_file`) and attaches a `SubagentUsage[]` to the returned `SessionUsage`. Each is parsed with `parse_file(.., force_context=true)` ‚Äî the `isSidechain` context gate in `process_line` would otherwise leave `context_tokens` at 0 since every line is sidechain; forcing it open tracks the subagent's **own** context window. `running` is a heuristic (file modified within ~8s). The frontend shows them as a thin strip of mini context-bar chips under the pane header (`SubagentStrip`/`SubagentChip` in `AgentPane.tsx`) ‚Äî kept separate from the parent's `ContextGauge`, since subagents have their own windows ‚Äî and a Subagents section in the stats popover.

Claude subscription limits (5h/weekly windows in the title bar) come from the Anthropic OAuth usage endpoint, authenticated with the local Claude Code login (macOS Keychain `Claude Code-credentials`, fallback `~/.claude/.credentials.json`) ‚Äî implemented in `src-tauri/src/limits.rs`, polled once a minute by `src/lib/limits.ts` (paused while hidden). A `null` result means "no usable Claude login" (missing credentials, or an expired/revoked token ‚Äî 401/403) and hides the Claude meters; transient fetch errors (network, 5xx) reject instead, and the store keeps the last known values ‚Äî preserve this distinction. Codex limit windows are read from active Codex `token_count.rate_limits` events and shown alongside Claude meters when present. Tokens/cost are tracked but only shown on demand (per-pane stats popover ‚Äî also in the ‚ãØ menu's Stats submenu for narrow panes ‚Äî and the usage drawer) ‚Äî not as raw token totals in pane headers or the title bar.

## Git status

Per-pane git status (repo, branch, ¬±lines, untracked, `origin` URL for "Open repo in browser") is strictly read-only and shells out to the git binary: `src-tauri/src/git.rs` (prefers the Settings `gitPath` override, else `/usr/bin/git` because GUI apps get a minimal PATH; every subprocess runs through `output_with_timeout` ‚Äî a repo on a dead network volume must not wedge the poll loop forever). `src/lib/git.ts` polls every 7 s, one fetch per distinct cwd, paused while the window is hidden; `null` on an agent means "checked, not a repo".

## Git worktrees

`src-tauri/src/worktree.rs` + `src/lib/worktree.ts`, native-only direct `invoke` like openrouter.ts ‚Äî deliberately *writing* git, unlike the read-only git.rs. The New Agent dialog shows a worktree toggle (default off) whenever the chosen cwd is in a git repo; enabling it reveals a branch field (random `repo/prefix-suffix-1234` name from `generateBranchName`, dice = reroll, manual input wins) and a "Copy environment" switch (default on). Submit is gated while the repo check is in flight (it must never silently downgrade to a plain pane), creates `<repo>/.worktrees/<slug>` via `git worktree add -b` off the current HEAD (main root resolved via `--git-common-dir`, so splitting from inside a worktree still lands in the main repo), excludes `.worktrees/` via `.git/info/exclude` (never the tracked `.gitignore`), copies every untracked file (`ls-files --others`, incl. gitignored `.env`s) minus the heavyweight `SKIP_DIRS` (node_modules, target, dist, ‚Ä¶), then spawns the agent with cwd = worktree path and `Agent.worktree = {root, branch}` (persisted in the grid snapshot ‚Üí restored panes keep their badge/lifecycle; tab auto-name/defaultCwd/lastCwd use the *root*, never the `.worktrees` path). Cancelling the dialog mid-create rolls the fresh worktree back instead of spawning an unwanted pane. Splits from a worktree pane prefill the toggle + a fresh branch on the root repo.

**Close lifecycle** (`requestRemoveAgent`): a pane with an agent actively working first raises `CloseBusyDialog.tsx` (Claude activity from OSC 9;4, Codex activity from rollout events; misclick guard mirrors the quit guard's rationale); then worktree panes check `worktree_status` ‚Äî clean (no uncommitted changes, `ahead == 0` where ahead = commits no other local branch/remote can reach) ‚Üí silently scheduled for deletion (folder `worktree remove --force` + `branch -D`, executed in `removeAgent` via the module-level `pendingWorktreeCleanup` map; a cancel **or detach** in the floating-terminal dialog aborts it ‚Äî a detached shell keeps the worktree as its cwd); dirty/unmerged ‚Üí `CloseWorktreeDialog.tsx` (keep / delete / cancel; keep = pane closes, worktree stays). Cleanup entries carry a `confirmed` flag: a silent clean verdict is **re-checked in `removeAgent` right before execution** (the decision can be minutes old after dialogs sat open ‚Äî on doubt the worktree is kept and shows as orphan), only an explicit dialog "delete" runs unconditionally. Gitignored files (e.g. the copied `.env`s) never count as dirty ‚Äî by design. Closing a whole workspace never deletes worktrees, and removing an agent clears any confirm dialog still pointing at it.

**Management panel** (`WorktreePanel.tsx`): a title-bar `FolderGit2` button, visible only while ‚â•1 worktree exists, lists all SwarmZ worktrees grouped per repo (open/orphaned/missing + dirty/ahead state, refreshed on open) with open-in-pane (or jump-to-pane), reveal-in-Finder (`opener:allow-reveal-item-in-dir` capability) and delete (two-step confirm when work would be lost; disabled while a pane is open on it). Scanned repo roots persist in `settings.worktreeRepos` (registered on create, pruned automatically once a repo has no worktrees left ‚Äî that's what makes the icon disappear again; `worktree_list` returns a `WorktreeScan {entries, scanned}` and only roots that actually **scanned** empty are pruned, against the current registry ‚Äî an unmounted volume or broken git override never loses orphans). A hand-deleted worktree folder still reports its branch's unreachable commits as `ahead` (computed from the main repo), so the panel's missing rows stay behind the two-step confirm when deleting would lose work.

## Panes, focus mode, floating terminals

Pane headers are container-query responsive (`@container` on the header in `AgentPane.tsx`): below ~36rem the folder path, gauge readout and split buttons collapse, below ~32rem the model badge, below ~28rem also the git chip, stats, mic and focus buttons ‚Äî only status dot, truncated title (`min-w-0 truncate`, never `shrink-0` ‚Äî it must shrink for the ellipsis to engage), context donut, ‚ãØ menu and close remain. Everything hidden stays reachable via the ‚ãØ menu (incl. the Stats submenu) or hotkeys (focus = header double-click, dictation = ‚åò hold/‚åò‚áßM).

Focus mode (`focusedAgentId` in the store, in-memory): the pane zooms (~150 ms) to an overlay above a dimmed backdrop in `TilingGrid.tsx` ‚Äî nothing unmounts, all other PTYs keep running underneath. Enter via header button / header double-click / ‚ãØ menu; exit via the same toggle or a backdrop click (no Escape ‚Äî the terminal eats it). Creating or removing agents, switching workspaces and opening the fleet all leave focus mode; pane drag-rearrange is disabled while focused. The exit animation relies on a render-phase `closingFocusId` in `TilingGrid.tsx` that keeps the transition class on the wrapper while the rect snaps back.

Floating terminals (`FloatingTerminals.tsx`): PiP-style plain shells above the grid, owned by an agent pane (same cwd), opened via header button / ‚ãØ menu / `‚åòJ`. In-memory, rendered in one layer inside `<main>`; windows stay mounted through minimize/detach and are only hidden (not unmounted) while their owner's workspace is inactive or the fleet is open ‚Äî removal from the store is what kills the PTY (`destroyTerm`); detached windows (`agentId: null`) show in every workspace. Stacking via a per-window `z` (never reorder the DOM ‚Äî xterm canvas). Windows name themselves after the last command run ‚Äî preset clicks and typed lines alike (`onCommand` on `TerminalView`, a best-effort input-line buffer that skips history-recall/escape-edited lines). Their quick-command bar mixes **presets per project folder** (persisted, store key `commandPresets` ‚Üí `FolderCommands { presets, hidden }`, keyed by cwd via `presetKey`; presets are editable in place, a preset matching a detected command's label or command **overrides** it, and detected commands can be hidden per folder + restored) with auto-detected commands: `project_commands` in `src-tauri/src/project.rs` (package.json scripts incl. lockfile‚Üípackage-manager mapping, cargo, Makefile/justfile targets; script order is alphabetical; script names are sanitized to a safe character set ‚Äî a name with control characters must never become a button that writes hidden shell lines). Closing a pane goes through `requestRemoveAgent`, which asks the backend whether a floating terminal still has child processes (`pty_has_children`, pgrep on the shell pid) and only then raises the kill-vs-detach dialog (`CloseAgentDialog.tsx`); detached terminals live on with `agentId: null`.

## Status reporting & quit guard

Pane title & live status come from runtime-specific signals. `Terminal.tsx` handles OSC 0/2 titles (Claude's auto-generated topic) and OSC 9;4 progress reporting into `Agent.activity` busy/idle (the status dot, the live per-tab `busy/total` counts, the fleet cards, and the quit guard). Claude Code only emits OSC 9;4 if the terminal advertises support ‚Äî the PTY backend sets `ConEmuANSI=ON` for that (deliberately, least invasive marker; don't remove). OSC 21337 (Claude's idle/busy/waiting tab-status pill) is pre-wired but dormant ‚Äî current Claude Code builds never emit it. Codex pane titles come from `~/.codex/session_index.jsonl` `thread_name`, and Codex busy/idle comes from `task_started` / `task_complete` rollout events parsed by `codex_usage.rs`.

Quit guard (`src/lib/quit.ts`, started in `App.tsx`): closing the app raises `QuitConfirmDialog.tsx` when quitting would lose something (`quitBlockerIds` + async `floatBlockerIds`) ‚Äî agents with `activity === "busy"` always count (the run gets interrupted, spinner in the list); idle-but-open panes count only while `restoreAgents` is off (with restore on they come back anyway, so they don't block); `exited` panes never count; floating terminals count when `pty_has_children` says a process still runs in them (floats are never restored). Native covers both paths ‚Äî window close via `onCloseRequested` (every first close is preventDefault'ed, all debounced persists are flushed (`flushAllPersists`), then `close()` is re-issued past a module-level `confirmed` flag so the window-state plugin still saves), and ‚åòQ/menu quit via Rust (`lib.rs` run-callback: `ExitRequested` with `code.is_some()` ‚Üí `prevent_exit()` + emit `app://quit-requested`; `prevent_exit` is a tauri-built-in no-op for the updater's `RESTART_EXIT_CODE`, and code `None` = last window closed passes, which is how a confirmed quit actually exits).

## OS file drag & drop

Tauri's default `dragDropEnabled` intercepts file drags at the NSView level, so the DOM never sees HTML5 drop events ‚Äî `src/lib/dnd.ts` (started in `App.tsx`) subscribes to the webview's `onDragDropEvent`, hit-tests the cursor via `elementFromPoint` against `[data-file-drop]` zones (the terminal containers in `AgentPane.tsx` and `FloatingTerminals.tsx`, attribute value = pty id) and on drop **pastes** the Terminal.app-style backslash-escaped path(s) (`escapeDropPath` in `utils.ts`) + a trailing space into that terminal via `term.paste()` (xterm instances are looked up in `lib/term-registry.ts`, registered by `TerminalView`), then focuses/raises the target. Paste, not a raw PTY write, is essential: `term.paste()` wraps the text in bracketed-paste markers when the app enabled mode 2004, which agent CLIs handle more reliably for file paths and multiline prompts than raw keystroke writes. While a drag is in flight, every zone shows `FileDropOverlay.tsx` (store key `fileDrag`, in-memory; `pointer-events-none` so the overlay stays invisible to the hit test); the hovered zone lights up `--ring` blue. Beware: the drag events' `position` is typed `PhysicalPosition`, but on macOS the wry ‚Üí tauri chain never scales NSView points, so the values are effectively logical CSS px ‚Äî use them unscaled.

## Performance invariants

PTY output is coalesced (‚â§12 ms / 128 KiB) and addressed per agent (`pty://data/<id>`); the usage file-watcher emits the changed project-dir names and the frontend skips refreshes for dirs it doesn't display, throttles to ‚â•2 s, and pauses while the window is hidden. Heavy Tauri commands (usage parsing, git subprocesses, keychain access, PTY kill, model load/inference) run async + `spawn_blocking` ‚Äî sync commands execute on the main thread and may only do fast, non-blocking work.

## Security

CSP is set in `tauri.conf.json` (no remote script/connect sources ‚Äî the webview talks IPC only). The OpenRouter key is keychain-ACLed to the SwarmZ binary (Security framework). The updater verifies minisign signatures against the pinned pubkey.
